# 数据出境路径分流搜索策略

## 何时读取本文件

当需要检索数据出境法规、申报指南、标准合同、认证规则、豁免规定或地方试点材料时，必须先读取本文件。

## 本文件影响什么决策

本文件决定官方 URL 速查、法律法规检索、官方域名搜索和高质量补充来源的执行顺序，以及搜索失败后的降级动作。

## 不适用场景

用户已经提供足够可定位原文、只需整理既有材料，或正在撰写报告格式时，不以本文件替代输出规则。

> 本文件是 `legato-china-data-export-pathway-triage` 自带的搜索策略指引。
> 目标是把"法规原文优先"的抽象原则翻译为 Agent 可逐步执行的具体搜索链路。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 搜索顺序

对每个子问题，Agent 必须按以下顺序执行搜索，不得跳过前序步骤直接泛搜：

### 第 1 步：official-source-verification.md 官方 URL 速查

查本 Skill 目录内 `references/official-source-verification.md` 的第 1 节"官方来源入口"，看该分流问题是否有已定位的官方来源 URL。

如有，优先直接访问该 URL 获取法规原文。已定位的官方 URL 覆盖：

- 数据出境安全评估办法（2022-07-07）
- 个人信息出境标准合同办法（2023-02-24）
- 标准合同附件（国务院公报）
- 个人信息保护认证实施规则（2022-11-18）
- 促进和规范数据跨境流动规定（2024-03-22）及国务院公报版
- 网络数据安全管理条例（2024-09）
- 数据出境安全评估申报指南第二版（2024-03-22）
- 天津自贸区负面清单（2024-05）

### 第 2 步：法律法规检索

对需要查证具体法条内容的分流问题，使用产品配置的法律法规检索能力在中国法规库中检索：

- 《个人信息保护法》（PIPL）相关条文
- 《数据安全法》相关条文
- 《网络安全法》相关条文
- 配套行政法规和部门规章

目标是找到法条原文中的具体门槛、条件和义务。

### 第 3 步：网信办官网搜索

在网信办官网中搜索最新通知、公告、解读和FAQ：

| 域名 | 用途 |
| --- | --- |
| `cac.gov.cn` | 网信办主站：数据出境相关通知、公告、办法、解读 |

搜索模板：

```
数据出境安全评估 site:cac.gov.cn
个人信息出境 标准合同 site:cac.gov.cn
数据出境 认证 site:cac.gov.cn
数据跨境流动 site:cac.gov.cn
```

### 第 4 步：法规库联网核验

在以下官方法规库中核验法规原文和最新修订：

| 域名 | 用途 | 说明 |
| --- | --- | --- |
| `flk.npc.gov.cn` | 国家法律法规数据库 | 法律、行政法规、部门规章原文 |
| `samr.gov.cn` | 国家市场监管总局 | 涉及认证规则的官方来源 |
| `gkml.samr.gov.cn` | 市场监管总局公开目录 | 认证认可相关规章 |
| `mof.gov.cn` | 财政部 | 涉及跨境数据的财政监管 |
| `miit.gov.cn` | 工信部 | 涉及行业数据分类的官方来源 |

使用域名限定搜索（如 `site:flk.npc.gov.cn`）在官方来源中核验。

### 第 5 步：高质量指南和权威解读搜索

官方来源无结果或覆盖不足时，在以下来源中搜索：

- DLA Piper Data Protection Laws of the World（免费，数据保护全球指南）
- ICLG - Data Protection - China
- Chambers - China Data Protection
- 全国信安标委 TC260 标准文本
- 权威法律期刊和数据库（如北大法宝、知网中的法规解读）

### 第 6 步：律所实务文章搜索

对时效性热点问题（新规解读、申报实操、退件经验），在律所网站或聚合平台中搜索：

- 优先使用 Lexology 作为发现工具
- 直接在主要律所官网搜索
- 中文律所（金杜、中伦、方达、汉坤、天元等）的数据合规团队文章

### 第 7 步：开放搜索

以上步骤均不足时，使用开放搜索补充。但结论强度自动降级，输出中必须标注为"待确认"。

## 2. 各步骤失败后的处理链

| 失败情况 | 下一步动作 | 输出要求 |
| --- | --- | --- |
| official-source-verification.md 无该场景覆盖 | 跳过，进入第 2 步 | 记录"该场景尚未收录官方来源，以下结论基于联网检索" |
| 法律法规检索无结果 | 进入第 3 步在网信办官网搜索 | 不因工具无结果就放弃法规原文查找 |
| 网信办官网搜索无结果 | 进入第 4 步在法规库核验 | 不写"该规则不存在"，继续搜索 |
| 官方来源 URL 失效（404/改版） | 换同域名其他入口或使用法规库核验 | 不因单个 URL 失效就放弃官方来源 |
| 法规库搜索无结果 | 不写"该规则不存在"，进入第 5 步 | 标注"官方来源未检索到，以下结论基于二手来源" |
| 高质量指南无覆盖 | 进入第 6 步或第 7 步 | 结论强度降为"待确认" |
| 仅找到律所解读文章 | 用作实务框架和操作参考 | 不作为决定性法律依据，标注"需法规原文核验" |
| 仅找到新闻或媒体 | 仅作为动态信号 | 结论标为"待确认"，不可单独支撑分流结论 |
| 来源之间存在冲突 | 比较效力层级、发布时间、适用范围 | 无法收口时标为"不明确"或"需咨询网信办" |
| 新旧法规冲突 | 比较生效日期和过渡期安排 | 标明"新规/旧规适用问题，需确认过渡期安排" |

## 3. 时效性前置判断

中国数据出境领域立法密集，启动研究时应先判断该子领域属于哪一级时效敏感度：

| 敏感度 | 典型领域 | 来源有效期窗口 | 行为要求 |
| --- | --- | --- | --- |
| 极高 | 数据出境新规动态（如《促进和规范数据跨境流动规定》后续修订）、自贸区负面清单更新、网信办最新通知 | 3-6 个月 | 在执行摘要中加醒目时效警告；推荐用户直接查阅官方最新来源；明确说明指南可能已过时 |
| 高 | 认证规则更新、标准合同备案实操变化、申报指南修订、重要数据目录发布 | 6-12 个月 | 注明所有来源的发布日期；关键结论建议交叉验证至少两个来源 |
| 中 | PIPL 基本框架、数据出境安全评估办法基本规则、标准合同办法基本规则 | 1-3 年 | 标准来源日期意识；使用指南时注明年份 |
| 低 | 法律体系结构、数据出境路径分类体系、法律层级关系 | 相对稳定 | 时效性不是关键问题，但仍应注明来源日期 |

对"极高"敏感度领域，报告的执行摘要和结论限制中必须包含：

> 中国数据出境领域规则变化频繁，新规与旧规可能存在过渡期安排。以下信息基于截至 [日期] 的来源，可能未反映最新变化。建议直接查阅网信办最新通知或咨询专业律师。

## 4. 分流判断中的搜索注意事项

数据出境路径分流是本 Skill 的核心输出。搜索时需特别注意：

### 阈值和门槛的精确性

- 安全评估触发条件中的数量级阈值（如 100 万人、敏感个人信息 1 万人）必须找到法规原文条款。
- 标准合同适用范围中的条件必须找到法规原文。
- 免申报条件必须找到法规原文中的具体表述。

### 新旧规则衔接

- 2024 年《促进和规范数据跨境流动规定》对 2022 年《数据出境安全评估办法》做了重要调整。搜索时必须同时覆盖两个文件，确认哪些条款被修改、哪些仍然有效。
- 关注网信办是否发布了过渡期安排或实施细则。

### 区域差异

- 自贸区负面清单仅适用于特定自贸区（目前仅天津已发布）。
- 搜索时需确认用户企业所在区域是否适用自贸区特殊规则。

## 5. 结论强度标注

根据搜索链路中的来源覆盖情况，在报告中标注结论强度：

| 情况 | 结论强度 |
| --- | --- |
| 有 official-source-verification.md 已定位官方 URL + 用户事实足够 | 可输出确定性较高的路径初判 |
| 有法规原文（PIPL/评估办法/标准合同办法/新规） + 用户事实足够 | 可输出决定性路径结论 |
| 有法规原文但用户事实不足 | 输出假设下的路径初判和事实缺口 |
| 仅有官方解读或权威指南 | 输出框架性结论，标注需法规原文核验 |
| 仅有实务文章 | 输出实务信号和操作参考，不输出法律结论 |
| 来源冲突或最新规则不明 | 降级为待核验，提示人工复核 |

## 6. 人工复核触发条件

以下情况必须触发人工复核提示（与 `official-source-verification.md` 第 5 节衔接）：

- 可能涉及重要数据、核心数据、国家秘密、政务数据。
- 可能涉及 CIIO 或大型平台（用户 1000 万以上）。
- 敏感个人信息或个人信息数量接近阈值（安全评估门槛）。
- 需要正式提交安全评估、标准合同备案或认证。
- 自贸区清单适用存在地区和行业差异。
- 仅找到实务文章或二手来源，未取得法规原文。
- 用户需要对外承诺、合同签署、监管沟通或正式法律意见。
- 新旧法规过渡期适用存在不确定性。
