# 中国数据出境路径与整改申报初筛：官方来源与实务边界

> 本文件供 `legato-china-data-export-pathway-triage` 运行时使用。  
> 本 Skill 独立打包，不依赖 `_shared` 或研究包素材总表。

## 何时读取本文件

- 开始任何中国数据出境路径初筛任务时。
- 涉及安全评估、标准合同、认证、豁免、自贸区负面清单、申报材料或整改降本时。

## 本文件影响什么决策

- 决定中国数据出境路径初判、官方来源优先级、企业实务参考边界和材料红旗。

## 不适用场景

- 不用于欧盟 GDPR / SCC / TIA / DPIA 风险判断。
- 不用于全球数据流动结构入口初筛。

## 2. 路径初筛框架

先按以下顺序判断：

1. 是否构成数据出境。
2. 是否符合免申报 / 豁免情形。
3. 是否涉及 CIIO、重要数据、敏感个人信息、大规模个人信息。
4. 是否落入安全评估、标准合同、认证或自贸区负面清单路径。
5. 是否可以通过整改降低出境必要性、范围、流程或残余风险。

## 3. 企业实务方法边界

| 企业实务方法 | 可用方式 | 禁止写法 |
| --- | --- | --- |
| 识别 -> 评估 -> 整改 -> 申报 | 用作分析顺序 | 不可写成官方法定流程的唯一顺序 |
| 四层整改框架 | 用于输出整改降本建议 | 不可作为监管批准依据 |
| 数据量三维描述 | 用于提示申报材料完整性 | 具体格式须以申报指南和监管要求为准 |
| 退件经验 | 用作实务红旗 | 不可写成所有项目必然退件条件 |

## 4. 结论强度

| 情况 | 结论强度 |
| --- | --- |
| 有官方来源 + 用户事实足够 | 可输出路径初判 |
| 有官方来源但用户事实不足 | 输出假设下的路径初判和事实缺口 |
| 仅有企业实务经验 | 输出实务红旗，不输出法律结论 |
| 来源冲突或最新规则不明 | 降级为待核验，提示人工复核 |

## 5. 人工复核触发

- 可能涉及重要数据、核心数据、国家秘密、政务数据。
- 可能涉及 CIIO 或大型平台。
- 敏感个人信息或个人信息数量接近阈值。
- 需要正式提交安全评估、标准合同备案或认证。
- 自贸区清单适用存在地区和行业差异。
- 用户需要对外承诺、合同签署、监管沟通或正式法律意见。
