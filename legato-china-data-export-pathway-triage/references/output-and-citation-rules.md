# 输出与引用规则（中国数据出境路径分流）

## 何时读取本文件

当准备输出中国数据出境路径初判、正式报告、引用法规依据或标注确定性等级时，必须先读取本文件。

## 本文件影响什么决策

本文件决定聊天摘要与正式 Markdown 报告的结构、法规引用方式、路径标签、证据缺口表达和结论降级规则。

## 不适用场景

仅做内部任务拆解、尚未进入用户可见输出，或只是在查找官方来源入口时，不以本文件作为主要检索指南。

> 本文件是 `legato-china-data-export-pathway-triage` 自带的输出与引用规则。
> 目标是确保数据出境路径分流场景下的所有判断依据有法规原文支撑、精度可控、时效性可追溯。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 输出层级

Legato 产品中，本 Skill 输出固定为"聊天摘要 + 强制正式 Markdown 交付物"：

1. 聊天窗口简短 Markdown 摘要，只帮助用户快速理解方向。
2. 正式 Markdown 报告交付物，是研究任务的强制交付面。

Markdown / Word 导出由产品交付物功能承担。

内部工具协议：

- 首轮研究任务必须使用 `write_file` 生成完整 Markdown 交付物，再使用 `present_artifacts` 展示。
- 同一研究的后续修改、补充、纠错、扩写或结构调整，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续对话如果是无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物，再使用 `present_artifacts` 展示。
- 纯解释既有交付物中某段含义、且不新增结论、依据、结构或行动清单时，可以只在聊天窗口回答。
- 工具名只用于 Skill 内部执行协议、eval 和研发说明，不得进入用户可见报告。

要求：

- 先给路径判断结论，再讲依据。
- 只放关键判断要素和限制。
- 不把完整分流报告塞进聊天消息。
- 不展示内部工具、供应商、配置编号或函数名。

## 3. 来源层级

中国数据出境路径分流场景的来源层级定义：

| 层级 | 来源 | 使用规则 |
| --- | --- | --- |
| 高 | 《个人信息保护法》《数据出境安全评估办法》《个人信息出境标准合同办法》《促进和规范数据跨境流动规定》《网络数据安全管理条例》、网信办通知公告 | 可支撑决定性分流结论 |
| 中 | 网信办申报指南、官方解读、权威法律指南（如 DLA Piper、ICLG 中国数据保护）、全国信安标委 TC260 标准 | 可辅助框架和操作判断 |
| 低 | 律所解读文章、行业报告、媒体快讯 | 只作动态信号或操作参考 |
| 待确认 | 未返回可定位片段、版本不明、来源冲突或事实缺失 | 必须提示人工复核 |

与 `official-source-verification.md` 的衔接：该文件第 1 节"官方来源入口"中列出的所有 URL 属于"高"层级来源，可直接引用。路径初筛框架（该文件第 2 节）和企业实务方法边界（该文件第 3 节）必须作为 Skill 输出的约束框架。

## 4. 确定性标签

报告中的每条数据出境信息必须标注确定性等级：

| 标签 | 含义 | 适用来源 |
| --- | --- | --- |
| [法规原文] | 直接引用法条或法规原文 | 高层级：PIPL 条文、安全评估办法、标准合同办法、促进和规范数据跨境流动规定原文 |
| [权威解读] | 来自网信办官方解读、TC260 标准、权威法律指南 | 中层级：官方解读和标准 |
| [一般评论] | 来自律所实务文章、行业分析、合规经验分享 | 中/低层级：律所文章、行业报告 |
| [待验证] | 来自 AI 生成、论坛、无法确认来源，或未找到官方核验 | 无来源或低可靠来源 |

### 分流类 Skill 的确定性标签特别要求

数据出境路径分流的核心是"判断依据 + 路径结论"。分流判断依据（如"达到 XX 数量级触发安全评估"）必须有法规原文支撑，否则不得输出确定性结论。

示例：

- 正确：根据《数据出境安全评估办法》第四条，"向境外提供 100 万人以上个人信息"触发安全评估 [法规原文]
- 错误：超过 100 万人需要做安全评估（未标注来源层级和确定性标签）
- 待确认：某解读文章提到"10 万人"可能是新门槛，但法规原文中未找到对应条文 [待验证]

## 5. 精度纪律

数据出境分流判断直接决定企业合规路径，精度要求极高：

- 源文件说"可能" / "可以"，不得转述为"必须" / "应当"。
- 源文件说"一般" / "通常"，不得转述为"一律" / "所有情形"。
- 法规中明确的数量级阈值必须引用原文条款，不得凭记忆或二手来源写阈值。
- 分流判断中的"免申报情形"必须列出法规原文中的具体条件，不得笼统写"可以豁免"。
- 阈值、门槛、有效期必须注明法规条文出处和生效日期。
- 多个来源说法不一致时（如 2024 年新规与旧规过渡期），分别列明，不揉成折中结论。
- 《促进和规范数据跨境流动规定》中"未被相关部门、地区告知或者公开发布为重要数据的，数据处理者不需要作为重要数据申报数据出境安全评估"等表述，必须按原文口径转述，不得扩大或缩小。

## 6. 脚注格式

优先使用 Markdown 脚注：

```markdown
正文中的关键法律结论[^1]。

[^1]: [来源标题](URL) - 高 | 法规原文 | 2026-05-25 | 说明该来源支撑的具体结论
```

数据出境场景的脚注示例：

```markdown
[^1]: [《数据出境安全评估办法》第四条](https://www.cac.gov.cn/2022-07/07/c_1658811536396503.htm) - 高 | 法规原文 | 2022-07-07 | 列明了四类强制安全评估情形
```

```markdown
[^2]: [《促进和规范数据跨境流动规定》第五条](https://www.cac.gov.cn/2024-03/22/c_1712776611775634.htm) - 高 | 法规原文 | 2024-03-22 | 规定了免予申报安全评估、订立标准合同、通过认证的情形
```

如果来源为权威解读：

```markdown
[^3]: [网信办官方解读 - 《数据出境安全评估办法》答记者问](URL) - 中 | 官方解读 | 2022 | 用于理解监管意图；具体门槛仍以法规原文为准
```

如果来源为律所解读：

```markdown
[^4]: [金杜 - 数据出境合规实务指南](URL) - 中 | 实务解读 | 2025 | 实务操作参考；法律结论仍需法规原文核验
```

## 7. 必须引用的内容

- 路径分流判断依据（安全评估触发条件、标准合同适用条件、认证适用条件、免申报条件）。
- 数量级阈值（如 100 万人个人信息、敏感个人信息 1 万人、重要数据认定）。
- CIIO（关键信息基础设施运营者）认定标准和义务。
- 重要数据的定义和认定规则。
- 免申报 / 豁免情形的具体条件（如自贸区负面清单、新规中的豁免条款）。
- 申报材料要求和申报流程（如安全评估申报指南第二版）。
- 标准合同备案要求和 PIA 义务。
- 认证证书有效期和认证依据。
- "当前有效""最新变化""过渡期安排"等时效判断。
- 多路径比较中的关键差异。
- 依赖二手来源或弱来源的判断。
- `official-source-verification.md` 第 2 节路径初筛框架。

## 8. 不得引用的内容

- 只有文件名、知识库名、网页标题，但没有可定位原文片段。
- 搜索摘要不足以支撑结论的内容。
- 推测性常识补全（如"数据出境一定要做安全评估"这类笼统表述）。
- AI 生成答案本身。
- 已废止或被新规覆盖的旧规则中的阈值和条件（除非在法规过渡期分析语境中）。

## 9. 冲突处理

来源冲突时：

- 分别列明冲突来源。
- 比较效力层级（法律 > 行政法规 > 部门规章/通知 > 标准 > 解读）。
- 特别注意新旧法规过渡期的冲突处理（如 2024 年新规与 2022 年评估办法的关系）。
- 如果仍无法收口，标为"不明确"或"需咨询网信办或专业律师"。

不要把冲突来源揉成一个看似确定的折中结论。

## 10. 翻译和原文

- 关键法律术语必须保留中文原文（如"数据出境安全评估""个人信息保护影响评估""标准合同"）。
- 法律名称使用官方全称，首次出现时标注简称。
- 英文翻译仅作参考，不替代中文法律术语。
