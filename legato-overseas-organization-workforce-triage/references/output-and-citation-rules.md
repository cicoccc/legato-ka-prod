# 海外组织与用工分流 — 输出与引用规则

## 何时读取本文件

- 准备输出海外团队与用工模式初筛报告、聊天摘要、引用脚注或分流建议前。
- 需要判断某条 EOR、工作许可、解雇、员工数据或 payroll 结论是否可写成确定结论时。

## 本文件影响什么决策

- 决定来源等级、分流结论强度、免责声明、人工复核触发和正式报告结构。

## 不适用场景

- 不用于寻找目标国官方入口或构造搜索 query；前者见 `official-source-entrypoints.md`，后者见 `search-templates.md`。

## 1. 输出层级

Legato 产品中，本 Skill 输出固定为"聊天摘要 + 强制正式 Markdown 交付物"：

1. 聊天窗口简短 Markdown 摘要，只帮助用户快速理解分流方向。
2. 正式 Markdown 报告交付物，是分流任务的强制交付面。

Markdown / Word 导出由产品交付物功能承担。

内部工具协议：

- 首轮分流任务必须使用 `write_file` 生成完整 Markdown 交付物，再使用 `present_artifacts` 展示。
- 同一研究的后续修改、补充、纠错或结构调整，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续对话如果是无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物，再使用 `present_artifacts` 展示。
- 纯解释既有交付物中某段含义、且不新增结论、依据、结构或分流建议时，可以只在聊天窗口回答。
- 工具名只用于 Skill 内部执行协议、eval 和研发说明，不得进入用户可见报告。

## 2. 聊天窗口回答

聊天摘要应像律师给客户提供建议——结论先行，用通俗语言说，语气自然。先让用户知道"这个国家/场景应该用什么用工模式"，再讲最关键的理由或限制。

底线要求：
- 必须包含分流方向（"建议用 EOR""需要设立本地实体""可以远程用工"等）。
- 必须说明已生成正式报告。
- 不得把完整报告内容塞进聊天消息。
- 不得暴露内部工具名、MCP ID 或供应商名称。

## 3. 来源层级

| 层级 | 来源 | 使用规则 |
| --- | --- | --- |
| 高 | 各国劳动法原文、劳动主管机关官方页面、移民局 / 签证机关官方页面、数据保护机关官方页面、ILO 官方数据库、政府公报 | 可支撑决定性分流判断（如"该国法律要求本地实体才能雇佣"） |
| 中 | ILO NATLEX 法规索引、ICLG Employment 指南、Chambers Employment 指南、Legal500 雇佣指南、Doing Business 系列、强实务文章、专家材料 | 可辅助框架和实务判断；关键分流依据仍需官方来源核验 |
| 低 | 律所新闻稿、媒体快讯、行业资讯、非正式评论 | 只作动态信号 |
| 待确认 | 未返回可定位片段、版本不明、来源冲突或事实缺失 | 必须提示人工复核 |

分流判断对来源层级的特殊要求：

- **"必须有本地实体才能雇佣"** 类判断：至少需要"高"层级来源支撑。
- **"EOR 模式在该国合法 / 不合法"** 类判断：至少需要"高"或"中"层级来源，且来源必须明确提及 EOR 或类似安排。
- **签证 / 工作许可时效判断**：来源日期距今超过 12 个月则必须标注"签证政策可能已变化，建议直接查阅官方最新来源"。

## 4. 确定性标签

报告中的每条分流判断依据必须标注确定性等级：

| 标签 | 含义 | 适用来源 |
| --- | --- | --- |
| [法规原文] | 直接引用法条或法规原文 | 高层级：官方法规库、政府公报、条约文本 |
| [权威解读] | 来自 ILO NATLEX、ICLG、Chambers、Legal500 等权威指南 | 中层级：高质量法律指南 |
| [一般评论] | 来自律所文章、新闻、行业分析 | 低层级：律所实务文章、媒体 |
| [待验证] | 来源不确定、无法确认、或未找到官方核验 | 待确认层级 |

使用规则：

- 分流判断的核心依据（如"该国法律是否允许 EOR"）必须标注 [法规原文] 或 [权威解读]，不得仅用 [一般评论] 支撑。
- 多个来源说法不一致时，分别列明各来源的确定性标签，不揉成折中结论。

## 5. 精度纪律

### 语义精度

- 源文件说 "may require"，不得转述为 "requires"。
- 源文件说 "in general" 或 "typically"，不得转述为 "always"。
- 源文件说 "can be" 或 "is possible"，不得转述为 "is guaranteed" 或 "is always permitted"。
- 源文件说 "subject to"，不得转述为 "unconditionally allowed"。
- 多个来源说法不一致时，分别列明，不揉成折中结论。

### 分流判断精度

- 分流结论必须区分"法律规定"与"实务惯例"。
- 不得将"某些企业使用 EOR"转述为"EOR 在该国完全合法"。
- 不得将"劳动法允许固定期限合同"转述为"所有岗位都可签固定期限合同"。
- 涉及"是否需要本地实体"的判断，必须区分"法律强制要求"和"实务上通常采用"。

## 6. 脚注格式

优先使用 Markdown 脚注：

```markdown
正文中的关键分流判断[^1]。

[^1]: [来源标题](URL) - 高 | 官方法规 | 2026-05-25 | 说明该来源支撑的具体分流结论
```

如果当前只取得高质量指南但尚未取得法条原文：

```markdown
[^1]: [ICLG Employment 指南](URL) - 中 | 专业指南 | 2026 | 用于搭建用工模式框架；关键分流依据仍需官方来源核验
```

如果来源为律所实务文章：

```markdown
[^1]: [律所实务解读](URL) - 低 | 律所文章 | 2026-05 | 仅作实务参考，不单独支撑分流判断
```

## 7. 必须引用的内容

- 决定性分流判断依据（如"该国要求本地实体才能雇佣"）。
- 签证 / 工作许可类别、申请条件、限制。
- 最低工资、社保费率、解雇保护期限等量化门槛。
- EOR / payroll / 劳务派遣模式在该国的法律地位。
- 外籍员工比例限制、劳动合同期限类型要求。
- "当前有效""最新变化""草案 / 已生效"等时效判断。
- 主管机关、办理路径、材料要求、官方入口。
- 多法域分流比较表中的关键差异。
- 依赖二手来源或弱来源的判断。

## 8. 不得引用的内容

- 只有文件名、知识库名、网页标题，但没有可定位原文片段。
- 搜索摘要不足以支撑分流结论的内容。
- 推测性常识补全（如"东南亚国家一般要求本地实体"）。
- AI 生成答案本身。
- 某律所的服务介绍页面（非实质法律分析）。

### 引用真实性底线

以下内容属于 LLM 模板占位符或幻觉产物，不得出现在任何引用、脚注或来源标注中：

- `[Valid]`、`[Effective]`、`[Current Date]`、`[现行有效]`、`[有效]`
- `[Body]`、`[X]`、`[Name]`、`[Date]`、`[数量]`、`[客户名称]`
- 未经实际检索验证的 URL 或域名（不得编造看起来合理但未实际访问的来源地址）
- 无对应脚注实体的纯数字标记（如 `[1]`、`[2]` 但文末无对应脚注条目）

引用必须来自实际检索命中或知识库召回的结果。不确定来源是否真实存在时，标为 [待验证] 而非伪造引用。

## 9. 冲突处理

来源冲突时：

- 分别列明冲突来源及其确定性标签。
- 比较效力层级（官方法规 > 指南 > 律所文章）、发布时间、适用范围和具体事实。
- 如果仍无法收口，标为"不明确"或"需当地确认"。

不要把冲突来源揉成一个看似确定的折中结论。

分流判断中来源冲突的特殊处理：

- 法规原文与指南说法不一致时，以法规原文为准。
- 不同指南说法不一致时，分别列明并标注"指南间存在差异"。
- 新旧法规衔接期间，明确标注"过渡期"并说明适用条件。

## 10. 翻译和原文

- 对非中文法域，关键术语尽量保留英文或当地官方语言原文。
- 用中文交付时，第一次出现原文术语后应解释含义（如"fixed-term contract（固定期限劳动合同）"）。
- 不把机器翻译或非官方译文当作无风险原文。
- 各国劳动法术语可能有当地特殊含义（如德国的 "Betriebsrat" 不简单等同于"工会"），需注明。

## 11. 分流类 Skill 特别规则

### 分流判断来源强制要求

以下分流判断必须有"高"或"中"层级来源支撑，不得仅基于低层级来源：

| 分流判断 | 最低来源层级 | 理由 |
| --- | --- | --- |
| 是否需要本地实体才能雇佣 | 高（法规原文或主管机关说明） | 直接影响用工模式选择 |
| EOR 模式在该国是否合法 | 中（权威指南或以上） | 法律合规核心问题 |
| 工作许可是否可由 EOR 代为申请 | 中（权威指南或以上） | 影响签证办理路径 |
| 固定期限合同最长年限 | 高（法规原文） | 合同设计关键约束 |
| 解雇保护的触发条件 | 高（法规原文） | 影响用工成本和风险 |
| 外籍员工比例限制 | 高（法规原文） | 影响团队配置 |

### 无法确认时的处理

- 官方来源搜索无结果：不得写"该国没有此规则"，应写"官方来源未检索到相关规定，以下结论基于二手来源，建议咨询当地律师确认"。
- 指南覆盖不足：标注"该法域在主流指南中覆盖有限，结论强度受限"。
- 用户要求用于实际用工决策：在报告中明确建议咨询当地劳动律师和 / 或移民律师。

### 来源不足时的降级说明

当整体来源覆盖不足时，报告执行摘要必须包含：

> 本分流结论基于截至 [日期] 的可用来源。部分法域的英文法律覆盖在主流指南中有限，具体分流建议需结合当地律师意见确认后方可在实际用工中执行。
