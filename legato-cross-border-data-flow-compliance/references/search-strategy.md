# 数据跨境合规搜索策略

## 何时读取本文件

当入口型数据合规研究需要检索中国、欧盟、行业监管、企业数据治理或跨境传输材料时，必须先读取本文件。

## 本文件影响什么决策

本文件决定法律法规检索、中国官方域名、欧盟官方来源、监管文件和开放搜索的先后顺序。

## 不适用场景

只是在判断是否转交中国数据出境或欧盟 TIA/DPIA 专精 Skill，或只是在撰写最终报告时，不以本文件替代方法边界和输出规则。

> 本文件是 `legato-cross-border-data-flow-compliance` 自带的搜索策略指引。
> 目标是把"官方来源优先"的抽象原则翻译为 Agent 可逐步执行的具体搜索链路。
> 与 `method-and-source-boundaries.md` 协同工作：来源边界见该文件，本文件聚焦搜索执行顺序和优先域名。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 通用搜索顺序

对每个子问题，Agent 必须按以下顺序执行搜索，不得跳过前序步骤直接泛搜：

### 第 1 步：法律法规检索

使用产品配置的法律法规检索能力直接检索中国法律条文：

- 《个人信息保护法》相关条文（第三十八条、第三十九条、第四十条等出境条款）
- 《数据安全法》相关条文（第三十一条、第三十六条等）
- 《网络安全法》相关条文（第三十七条等）
- 《数据出境安全评估办法》全文
- 《个人信息出境标准合同办法》全文
- 相关 GB 标准（GB/T 35273 等）

目标是取得法条原文和官方出处，为后续分析提供决定性依据。

### 第 2 步：中国官方域名限定搜索

从优先域名列表中使用域名限定搜索（`site:xxx.gov.cn`），在官方来源中检索：

| 优先域名 | 来源 | 搜索重点 |
| --- | --- | --- |
| `cac.gov.cn` | 国家互联网信息办公室 | 数据出境安全评估、标准合同、认证、FAQ |
| `flk.npc.gov.cn` | 国家法律法规数据库 | 法律法规原文 |
| `miit.gov.cn` | 工业和信息化部 | 数据安全、网络安全相关通知和办法 |
| `mps.gov.cn` | 公安部 | 网络安全等级保护、数据安全 |
| `samr.gov.cn` | 市场监管总局 | GB 标准相关 |

搜索模板：

```
数据出境 site:cac.gov.cn
个人信息出境安全评估 site:cac.gov.cn
标准合同 site:cac.gov.cn
数据安全评估 site:cac.gov.cn
数据出境安全评估办法 site:cac.gov.cn
```

### 第 3 步：欧盟官方来源搜索

如问题涉及 GDPR 跨境传输，使用域名限定搜索在欧盟官方来源中检索：

| 优先域名 | 来源 | 搜索重点 |
| --- | --- | --- |
| `edpb.europa.eu` | 欧洲数据保护委员会 | TIA 指南、DPIA 指南、SCC 相关建议 |
| `eur-lex.europa.eu` | 欧盟法规数据库 | GDPR 条文、adequacy decision |
| `ec.europa.eu` | 欧委会 | SCCs 官方文本、数据保护 Q&A |
| `commission.europa.eu` | 欧委会 | 跨境传输机制、adequacy decision |

搜索模板：

```
data transfer GDPR adequacy site:edpb.europa.eu
standard contractual clauses site:edpb.europa.eu
transfer impact assessment site:edpb.europa.eu
```

### 第 4 步：高质量法律指南搜索

官方域名无结果或覆盖不足时，在以下高质量法律指南中搜索：

- DLA Piper Data Protection（数据隐私专属数据库，覆盖面广）
- Chambers Practice Guides
- ICLG Data Protection
- Legal500 Country Guides
- IAPP（专业协会报告和分析）

搜索模板：

```
cross-border data transfer site:iclg.com
data protection {country} site:dlapiperdataprotection.com
data transfer {country} site:practiceguides.chambers.com
```

这些来源用于搭框架、做比较和识别需要官方核验的关键点，不替代官方来源。

### 第 5 步：律所实务文章搜索

对时效性热点问题（近期数据出境新规、新安全评估动态、SCC 修订），在国际所和本地强所的网站或聚合平台中搜索 client alert / legal update。

- 优先使用 Lexology 作为发现工具
- 直接在主要律所官网搜索
- 中文律所：金杜、中伦、方达、汉坤等数据合规团队

### 第 6 步：开放搜索

以上步骤均不足时，使用开放搜索补充。但结论强度自动降级，输出中必须标注为"待确认"。

## 2. 各步骤失败后的处理链

| 失败情况 | 下一步动作 | 输出要求 |
| --- | --- | --- |
| 法律法规检索无法返回完整条文 | 跳过，进入第 2 步 | 记录"法条检索未返回完整条文" |
| 官方域名 404 或改版 | 换同来源其他官方域名，再试一次 | 不因单个域名不可用就放弃官方来源 |
| 中国侧官方域名限定搜索无结果 | 不写"该规则不存在"，进入第 3 步或第 4 步 | 标注"官方来源未检索到，以下结论基于二手来源" |
| 欧盟侧 EDPB/EUR-Lex 无结果 | 进入第 4 步 | 标注"EDPB/EUR-Lex 未检索到" |
| 高质量指南无相关覆盖 | 进入第 5 步或第 6 步 | 结论强度降为"待确认" |
| 开放搜索仅找到中文文章 | 用作问题理解框架 | 不作为决定性依据，标注"仅找到中文解读，需官方来源核验" |
| 开放搜索仅找到新闻或媒体 | 仅作为动态信号 | 结论标为"待确认"，不可单独支撑法律结论 |
| 来源之间存在冲突（中国侧 vs 欧盟侧） | 分别列明，比较效力层级 | 无法收口时标为"不明确"，不揉成折中结论 |

## 3. 时效性前置判断

数据跨境合规属于时效性敏感度"极高"的领域：

| 敏感度 | 典型子领域 | 来源有效期窗口 | 行为要求 |
| --- | --- | --- | --- |
| 极高 | 数据出境新规、安全评估办法修订、SCC 修订、新的 adequacy decision | 3-6 个月 | 在执行摘要中加醒目时效警告；推荐用户直接查阅官方最新来源 |
| 高 | 评估办法实施细则、认证规则、行业数据出境指南 | 6-12 个月 | 注明所有来源的发布日期；关键结论交叉验证至少两个来源 |
| 中 | GDPR 基本框架（Chapter V）、PIPL 出境基本条款 | 1-3 年 | 标准来源日期意识；注意法条是否有修正案 |
| 低 | 法律体系结构、数据保护基本原则 | 相对稳定 | 仍应注明来源日期 |

报告的执行摘要和结论限制中必须包含：

> 本领域规则变化频繁（中国数据出境规则近年密集出台实施细则）。以下信息基于截至 [日期] 的来源，可能未反映最新变化。建议直接查阅网信办最新通知或咨询数据合规律师。

## 4. 中国侧特殊搜索注意事项

- 网信办政策经常以"征求意见稿"形式先行发布，需确认是已正式生效还是仍在征求意见阶段。
- 《数据出境安全评估办法》《个人信息出境标准合同办法》等配套规则可能有更新的实施细则或 FAQ。
- 部分行业（金融、医疗、汽车、AI）可能有额外的行业监管要求，需在通用规则之外补充行业搜索。
- 搜索时应区分"数据出境安全评估"（针对达到门槛的场景）和"个人信息出境标准合同"（针对未达门槛的场景），两者适用条件不同。

## 5. 欧盟侧特殊搜索注意事项

- GDPR Art.44-49 是跨境传输的基础框架，但实际操作需结合 EDPB 指南和 DPA 裁决。
- TIA 不是一次性任务，需要持续监控目的地国法律变化。
- Schrems II（C-311/18）之后，对美国传输有特殊的 adequacy decision（EU-US Data Privacy Framework），需确认当前状态。
- 涉及中国传输时，TikTok DPC 裁决（IN-21-9-2）是重要参考案例。
