# 中企出海数据流动合规初筛：专家方法与来源边界

> 本文件供 `legato-cross-border-data-flow-compliance` 运行时使用。  
> 本 Skill 独立打包，不依赖 `_shared` 或研究包素材总表。

## 何时读取本文件

- 开始任何出海数据流动风险初筛任务时。
- 需要判断是否属于全球数据流动入口问题，或是否应转中国数据出境 / 欧盟数据传输专精 Skill 时。

## 本文件影响什么决策

- 决定三明治结构识别、专家材料边界、转交条件、来源等级和人工复核触发。

## 不适用场景

- 不用于深做中国数据出境路径、GDPR / SCC / TIA / DPIA 或监管调查应对。

## 1. 专家心法保真

| 专家心法 | 可产品化表达 | 不能外推的边界 |
| --- | --- | --- |
| 三明治结构 | 优先识别“多国收集 -> 中国分析中枢 -> 多国使用”的结构性风险 | 不能把所有数据出境都归为三明治结构 |
| 合规跟随产品 / 交易设计 | 先还原业务和数据流，再判断规则与行动 | 不输出通用隐私合规清单冒充结论 |
| PIA / 数据合规动态触发 | 新产品、新市场、新处理活动可能重新触发评估 | 不替代正式 PIA / DPIA 报告 |
| 分层交付 | 管理层看风险雷达，法务看依据和缺口，业务看行动清单 | 不把管理层摘要当作完整法律意见 |
| AI 只做初筛 | 标注初筛性质，高风险场景触发人工复核 | 不承诺 100 分专业答案或最终法律意见 |

## 2. 材料来源边界

| 材料类型 | 可进入本 Skill 的方式 | 禁止写法 |
| --- | --- | --- |
| 匿名专家访谈、访谈规划、萃取报告 | 专家心法、任务边界、报告语言 | 不可把未确认推测写成专家结论 |
| 企业数据出境实务材料 | 仅可作为转交 Skill 2 的背景提示 | 不可写成本 Skill 的专家心法 |
| GDPR / DPC / CJEU / EDPB 材料 | 仅可作为转交 Skill 3 的背景提示 | 不在本 Skill 内深做 GDPR 结论 |
| 全球数据流通趋势资料 | 研究包背景和法域趋势参考 | 不作为决定性法律依据 |
| IPO 数据合规材料 | 第一版只做红旗扫描 | 不输出问询回复或上市障碍最终结论 |

## 3. 路由规则

| 用户问题特征 | 本 Skill 行为 |
| --- | --- |
| 全球业务、多国收集、数据回中国总部分析 | 本 Skill 主承接 |
| 不知道从中国出境还是 GDPR 还是目标国规则入手 | 本 Skill 主承接并给转交建议 |
| 明确问安全评估 / 标准合同 / 认证 / 豁免 | 转 `legato-china-data-export-pathway-triage` |
| 明确问 GDPR / SCC / TIA / DPIA / EU 远程访问 | 转 `legato-eu-data-transfer-tia-dpia-triage` |
| 明确问 PIPL 第 38 条原文等法条 | 转 `legato-china-legal-research` |
| 明确问劳动、税务、组织、德国运营 | 转对应专家 Skill |

## 4. 引用纪律

- 专家方法引用必须能定位到匿名专家材料。
- 法律规则引用必须来自官方来源或高质量公开来源。
- 如果只是路由建议，不要强行挂法律引用。
- 无可定位原文片段不得挂引用编号。

## 5. 人工复核触发

- 涉及 3 个以上法域的规则冲突。
- 敏感行业：金融、医疗、电信、车联网、AI 训练数据、未成年人。
- 涉及敏感个人信息、重要数据或大规模数据。
- 涉及 IPO、并购、融资、客户重大审计。
- 已出现监管调查、投诉、处罚、诉讼或数据泄露。
- 用户要求正式法律意见、对外披露或提交监管材料。
