# 出海法律与合规研究 Skill

> 版本：v0.4  
> 日期：2026-05-31  
> 状态：productized  
> 运行模式：v0.1 固定 `agent`

## 定位

本 Skill 是 Legato 知识助手中的出海法律与合规研究通用兜底能力，用于中国企业在境外市场进入、跨境销售、主体设立、海外用工、监管许可、数据合规、投资路径、争议风险和多法域比较中的初步法律研究。

它解决“去哪个法域查、查什么类型来源、如何判断结论强度、何时提示人工复核”的问题，不替代当地律师最终法律意见。

## 适用场景

- 查某个国家 / 地区的法律规则、监管限制、许可或备案要求。
- 比较多个法域的设立、准入、劳动、数据、税务、争议解决或合规路径。
- 为中国企业出海做管理层摘要、风险扫描、行动清单或正式 Markdown 报告。
- 用户资料不足时，先形成有限研究框架并标明需要补充的目标法域、业务事实和材料。

## 不适用边界

- 主问题是中国大陆法，应转中国法规案例研究 Skill。
- 主问题是德国投资运营税务与合规，应优先转德国专家 Skill。
- 主任务是合同起草 / 审查、诉讼代理、企业工商尽调、翻译、排版或纯市场研究。
- 用户要求最终法律意见时，只能输出初筛、依据、限制和当地律师复核提示。

## 最低输入

最低可启动输入：

- 目标法域，或可从业务区域合理推断的法域。
- 主题：市场进入、主体设立、劳动、税务、数据、许可、争议解决、多法域比较等。
- 基本业务事实：主体、行业、交易、产品、服务、人员或合规问题。

法域不明确时，不伪造目标国家；可输出有限框架，并说明需要补充的法域会显著影响结论。

## 研究执行顺序

1. 先识别法域，再识别主题，再按 `references/source-routing.md` 的主题-专业数据库速查表选择搜索策略。
2. 按 `references/search-strategy.md` 中的五步搜索顺序执行：专业数据库 → 官方域名限定搜索 → 高质量法律指南 → 律所实务文章 → 开放搜索；不得跳过前序步骤直接泛搜。
3. 使用 `references/search-templates.md` 中的查询模板构造搜索，不自行生成泛化查询。
4. 启动研究时先按 `references/search-strategy.md` 做时效性前置判断，按领域敏感度决定警惕等级和来源有效期评估。
5. 对每条法律信息标注确定性标签（法规原文 / 权威指南 / 一般评论 / 待验证），源文件说 "may require" 不得转述为 "requires"。
6. 联网搜索到指南类内容时，只提取目录和章节标题，不概括其中法律条文；需要法条时从指南中找引用，再回官方来源查原文。
7. 按本轮检索结果标注覆盖状态：强 / 中 / 弱 / 未覆盖；不得把来源包中存在域名直接等同于强覆盖。
8. 多法域比较必须统一维度，不把某一国规则泛化到其他法域。
9. 来源不足、版本不明、冲突或无法定位片段时，结论降级为待核验。
10. 复杂路径选择问题可输出策略路径对比表；法条定位、定义解释或单点规则确认不触发该表。

| 触发情况 | 必读文件 | 读取目的 | 读后动作 |
| --- | --- | --- | --- |
| 开始任何出海法律与合规研究 | `references/source-routing.md` | 确定法域、主题类别、优先来源和专业数据库入口 | 再选择检索路径 |

| 准备输出聊天摘要或正式报告 | `references/output-and-citation-rules.md` | 检查引用、脚注、来源等级和结论强度 | 再生成交付物 |

## 任务分型

| 类型 | 主路径 | 输出重点 |
| --- | --- | --- |
| 规则存在性 | 官方法规 / 主管机关 | 是否存在、适用条件、来源强度 |
| 操作路径 | 政府办事指南 / 监管 FAQ | 步骤、材料、时间、责任主体 |
| 多法域比较 | 同一维度 + 各法域来源 | 比较表、差异、结论强度 |
| 法律文件解读 | 文件原文 + 背景来源 | 条款结构、义务、风险 |
| 风险扫描 | 法域 + 主题 + 项目资料 | 红旗、优先级、人工复核 |
| 动态追踪 | 官方公告 + 时间标记 | 现行规则、草案、趋势 |

## 运行约束

- `mode = agent`。
- 不调用 `ask_user_question`。
- 不使用旧 `query_kb`。
- 不自行选择知识库、文件夹或文件范围；检索范围由前端挂载和研究包配置决定。
- 联网检索只表达地域、语言、复杂度、可信源要求、域名 / 时间约束和来源优先级；具体搜索工具选择遵循系统级搜索工具选择策略。
- 本 Skill 独立打包，来源入口使用本目录内 `references/source-routing.md`，不依赖外部共享目录。
- 首轮研究任务必须使用 `write_file` 生成完整 Markdown 交付物，并使用 `present_artifacts` 展示；不得因问题较窄、只需简短答复或只是流程型检索而省略交付物。
- 同一研究的后续修改、补充、纠错或结构调整，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续对话如果是无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物，再使用 `present_artifacts` 展示。
- 不固定底层输出路径或文件名模板，由产品运行时处理落盘位置。

## 引用与复核规则

- 关键法律结论必须有官方来源或高质量来源的可定位片段。
- 无可定位原文片段不得挂引用编号。
- 不接受只有 `[domain]` 的域名级引用；引用必须指向具体页面、文件、法规 / 规则名、条号或章节、日期和支撑点。
- 输出前过滤 `[Valid]`、`[Current Date]`、`[X]`、`[有效]`、无脚注实体的纯数字标记等伪引用。
- 高质量指南、律所文章和新闻不能单独支撑决定性结论。
- 涉及当地律师执业判断、监管审批、处罚、诉讼、移民、税务、数据跨境或劳动争议时，必须提示人工复核。

## 输出要求

聊天窗口只放结论、关键依据、风险限制、下一步和交付物状态；聊天摘要不能替代正式交付物。任何出海法律与合规研究首轮任务都必须生成完整 Markdown 交付物，包括单点流程检索、单法域概览和多法域比较。

正式报告必须先读取并使用 `assets/outbound-legal-report-template.md`作为输出模板，调用`write_file`生成交付物文件。同一研究的后续修改用 `edit_file` 更新原报告；无关新研究新建交付物。来源路由和引用细则按 Reference 使用路由中的 `references/source-routing.md`、`references/output-and-citation-rules.md` 执行。
