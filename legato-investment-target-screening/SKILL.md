# 投资标的法律快筛 Skill

> 版本：v0.4
> 日期：2026-05-31
> 状态：productized
> 运行模式：v0.1 固定 `agent`

## 定位

本 Skill 是 Legato 知识助手中投资标的法律快筛能力，用于在投资决策早期快速出具目标公司的主体可信度、法律红旗和行动建议。

与通用企业数据工具的区别：不只是搬运数据填模板，而是以法律视角解读数据——判断"这些数据对本次投资交易意味着什么，需要在交易文件中设定哪些保护条款"。

详细方法论见：`references/expert-method.md`。

## 适用场景

- PE/VC 项目立项前的快速法律筛查。
- 企业战投对标的公司的初步背调。
- Term Sheet 前的法律风险快扫。
- 投资经理批量浏览项目池时的初筛。
- LP 推介前的企业基本面了解。

## 不适用边界

- 完整尽调报告（IC Memo），需要 6 Server 全量调用和 10 章报告，应由更重的 Skill 或专项 DD 承担。
- 竞品对比分析，需要多企业输入。
- 供应商准入评估、年度健康体检，属采购供应链场景。
- 融资历史深度追踪与估值分析。
- 合同审查、交易文件起草、诉讼策略。
- 境外企业筛查，本 Skill 覆盖中国境内企业。

## 最低输入

最低可启动输入：

- 目标公司名称或统一社会信用代码。

可选输入（提高结论精度）：

- 投资轮次（Pre-A / A / B / C / Pre-IPO）。
- 投资类型（股权投资 / 资产收购 / 增资 / 合营）。
- 投资主题或关注重点（如"主要看知产""创始人背景很重要"）。

信息不足时不调用即时追问工具；基于已有信息推进，在报告中标注假设和限制。

## 研究执行顺序

本 Skill 按六个维度依次扫描，每个维度选择最少必要工具。详细判断规则、红旗清单和风险解读框架由 `references/expert-method.md` 承载。

1. 按 `Reference 使用路由` 读取专家方法、MCP 路由和引用规则，先确认扫描维度、工具边界和停止条件。
2. **主体核验**：确认目标公司工商登记信息、登记状态和治理稳定性。
3. **股权与实控人**：穿透股权结构，识别实控人、UBO、代持风险和 VIE 架构。
4. **核心风险扫描**：快扫失信、限高、被执行人、股权冻结、经营异常、行政处罚等风险信号。
5. **管理层画像**：对实控人和法定代表人做 4 项核心红线快扫（失信/限高/被执行/限出境）。
6. **知产资产概览**：统计专利、商标、软著数量，检查知产出质情况。
7. **法律红旗综合**：基于上述数据，生成投资交易法律红旗清单和建议行动。

每个维度完成后汇总风险标签，最终生成红绿灯结论。

| 触发情况 | 必读文件 | 读取目的 | 读后动作 |

| 准备输出报告或引用事实 | `references/output-and-citation-rules.md` | 检查数据引用、未披露字段、结论强度和输出边界 | 再生成交付物 |

## 运行约束

- `mode = agent`。
- 不调用 `ask_user_question`。
- 不使用旧 `query_kb`。
- 不自行选择知识库、文件夹或文件范围。
- 内部工具名只允许出现在运行配置、研发排查或 eval 中，不进入用户可见报告。
- 首轮研究任务必须使用 `write_file` 生成完整 Markdown 交付物，并使用 `present_artifacts` 展示。
- 同一研究的后续修改、补充或纠错，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物。
- 不固定底层输出路径或文件名模板。
- MCP 数据只写从工具返回的内容，不编造、不补全、不推测。MCP 没返回的字段标注"未披露"。

## MCP 依赖

| MCP ID | 界面名称 | 用途 | 是否必需 |
| --- | --- | --- | --- |
| `qcc-company` | 企业基座 | 工商核验、股东、实控人、UBO、财务数据 | 是 |
| `qcc-risk` | 风控大脑 | 失信、限高、被执行、股权冻结、经营异常、行政处罚 | 是 |
| `qcc-executive` | 董监高画像 | 实控人和法代个人信用红线扫描 | 建议 |
| `qcc-ipr` | 知产引擎 | 专利、商标、软著、知产出质 | 建议 |
| `qcc-operation` | 经营罗盘 | 融资记录、招聘活跃度、荣誉 | 可选 |
| `qcc-history` | 历史存档 | 治理稳定性（历史法代、历史股东、历史工商变更） | 可选 |
| `yuandian-law` | 法律法规 | 按需检索投资场景相关法规（如对赌协议效力、股权代持规定） | 按需 |

MCP 工具调用总计约 12-15 次，控制成本和响应时间。详细工具映射由 `references/mcp-routing.md` 承载。

## 联网搜索

联网默认开启。主要用于：

- 核验目标公司最新新闻、舆情、监管处罚。
- 搜索行业最新监管政策和投资环境变化。
- 补充 MCP 数据未覆盖的公开信息。

具体搜索工具选择遵循系统级搜索工具选择策略；本 Skill 只表达国内 / 公司信息 / 新闻时效 / 官方来源 / 交叉验证等检索意图和来源优先级。

## 输出要求

### 聊天窗口回答

一句话结论 + 红绿灯状态 + 3-5 个关键发现 + 正式报告状态 + 限制提示。

### 正式 Markdown 报告

首轮必须生成 8 章结构报告。报告模板见 `assets/investment-screening-report-template.md`。

## 免责与边界

- 所有分析基于企业公开数据，不构成投资建议或法律意见。
- 本 Skill 输出仅供初步筛查，不得替代完整尽调（IC Memo + KYB + 专项 DD）。
- `get_financial_data` 对非上市公司覆盖有限，早期项目大概率返回空。
- 最终投资决策由投委会综合评审。
