# 投资标的快速法律筛查 — 专家方法论

---

## 何时读取本文件

- 开始任何投资标的法律快筛任务时。
- 需要确定六维扫描顺序、红旗条件、风险解读框架或停止条件时。

## 本文件影响什么决策

- 决定主体核验、股权穿透、风险扫描、管理层画像、知产概览和法律红旗综合的执行顺序。

## 不适用场景

- 不用于决定 MCP 工具调用细节；工具路由见 `mcp-routing.md`。
- 不用于报告引用和未披露字段表达；输出规则见 `output-and-citation-rules.md`。

> 本文件中出现的 MCP 工具函数名（如 `get_company_registration_info`、`get_dishonest_info` 等）仅供 Agent 执行参考，不得进入用户可见报告、脚注或来源说明。

## 1. 六维扫描工作流

### 维度一：主体核验

目标：确认目标公司合法存续、登记状态正常。

| 检查项 | MCP 工具 | 红旗条件 |
| --- | --- | --- |
| 工商登记信息 | `get_company_registration_info` | 登记状态非"存续"或"在营" |
| 企业准确性校验 | `verify_company_accuracy` | 名称/USCC 不匹配 |
| 股东信息 | `get_shareholder_info` | 股东数量异常少或代持迹象 |
| 实际控制人 | `get_actual_controller` | 实控人不明或与股东不一致 |
| 受益所有人 | `get_beneficial_owners` | UBO 模糊或多层嵌套 |

**治理稳定性判断**（需 qcc-history）：
- 查历史法代变更、历史注册资本变更、历史名称变更。
- 3 年内法代变更 ≥ 3 次 → 不稳定。
- 5 年内法代变更 ≥ 5 次 → 高度不稳定。

### 维度二：股权结构与实控人

目标：识别股权清洁度、代持风险、VIE 架构。

**穿透规则**：
- 遇到合伙企业 / 持股平台 / SPV，追加 `get_shareholder_info` 穿透，最多 3 层。
- 标注股权结构复杂度：简单 / 中等 / 复杂。

**法律关注点**：
- 代持安排：股东与实控人不一致时提示代持可能性。
- VIE 架构：涉及协议控制时标注，影响投资可行性判断。
- 一致行动人：多个自然人股东间是否存在一致行动安排。

### 维度三：核心风险扫描

目标：快扫 6 类核心风险信号。

| 风险类型 | MCP 工具 | 红旗条件 | 法律影响 |
| --- | --- | --- | --- |
| 失信被执行人 | `get_dishonest_info` | 有记录 | 重大负面：信用崩塌，投资退出困难 |
| 限制高消费 | `get_high_consumption_restriction` | 有记录 | 重大负面：实控人或法代被限高 |
| 被执行人 | `get_judgment_debtor_info` | 有大额执行标的 | 负面：或有负债风险 |
| 股权冻结 | `get_equity_freeze` | 有记录 | 重大负面：股权无法正常交割 |
| 经营异常 | `get_business_exception` | 有未移出记录 | 负面：经营合规问题 |
| 行政处罚 | `get_administrative_penalty` | 有记录 | 需评估严重程度和行业影响 |

**进一步风险**（按需调用）：
- 破产重整：`get_bankruptcy_reorganization`。
- 严重违法：`get_serious_violation`。
- 股权出质：`get_equity_pledge_info`。
- 欠税公告：`get_tax_arrears_notice`。
- 税务异常：`get_tax_abnormal` / `get_tax_violation`。

**数据纪律**：
- MCP 返回什么就写什么，不编造处置状态。
- 不写"已处置""已闭环"等 MCP 未显式返回的结论。
- 处罚日期、处罚机关、决定书文号必须直引 MCP 原文。

### 维度四：管理层信用画像

目标：对实控人和法定代表人做 4 项核心红线快扫。

| 红线项 | MCP 工具 | 触发 = 重大风险 |
| --- | --- | --- |
| 失信 | `get_personnel_dishonest` | 实控人/法代是失信被执行人 |
| 限高 | `get_personnel_high_consumption_ban` | 被限制高消费 |
| 被执行 | `get_personnel_judgment_debtor` | 有未了结执行案件 |
| 限出境 | `get_personnel_exit_restriction` | 被限制出境 |

**判断规则**：
- 实控人和法代非同一人时，各扫一遍。
- 4 项全绿 → 通过。
- 任一触发 → 标红，需在报告中展开说明。
- 如果实控人信息获取不到，标注"实控人未明确识别，管理层画像仅覆盖法定代表人"。

### 维度五：知产资产概览

目标：统计核心知产数量，识别知产出质风险。

| 检查项 | MCP 工具 | 关注点 |
| --- | --- | --- |
| 专利 | `get_patent_info` | 总数、发明占比、核心技术领域 |
| 商标 | `get_trademark_info` | 总数、核心类别 |
| 软著 | `get_software_copyright_info` | 总数（软件企业重要指标） |
| 知产出质 | `get_ipr_pledge` | 出质比例 > 50% → 重大红旗 |

**法律影响**：
- 知产出质 > 50%：无形资产已大量抵押融资，净资产水分大，估值需调减。
- 零知产但自称科技公司：商业模式可信度存疑。

### 维度六：法律红旗综合

目标：基于前五个维度的数据，生成投资交易法律红旗清单。

**红旗清单逻辑**：

| 数据信号 | 法律红旗 | 建议行动 |
| --- | --- | --- |
| 失信/限高/被执行 | 实控人信用风险 | TS 中设陈述保证条款 + 交割前解除条件 |
| 股权冻结 | 股权交割障碍 | 要求交割前解除冻结 |
| 代持迹象 | 股权清晰度风险 | 要求代持还原并确权 |
| VIE 架构 | 投资结构风险 | 专项法律尽调 + 合规确认 |
| 知产出质 > 50% | 无形资产虚高 | 估值调减 + 要求解押 |
| 频繁法代变更 | 治理不稳定 | 加强交割后治理安排 |
| 行政处罚 | 合规风险 | 评估处罚影响 + SPA 赔偿条款 |
| 实控人多项控制企业 | 关联交易风险 | 关联交易披露和审批机制 |

---

## 2. 停止条件

可以停止：

- 已完成 6 个维度的 MCP 数据收集。
- 已识别所有红旗信号并给出法律影响和建议行动。
- 已按 `assets/investment-screening-report-template.md` 生成完整正式报告。

不能停止：

- MCP 返回错误但未重试。
- 实控人未识别但未说明。
- 红旗信号未给出法律影响解读。

---

## 3. 禁止路径

- 不编造 MCP 未返回的数据。
- 不写"已处置""已闭环"等结论性状态。
- 不因数据量大而省略股东、投资等列表的中间项。
- 不把"零风险"等同于"合规"——可能是企业太年轻。
- 不在画像速览层面给出投资建议，只给筛查结论和行动建议。
- 不对非上市公司财务数据做精确推算。

---

## 4. 人工复核触发条件

以下情况必须在报告中标注需人工复核：

- 实控人无法通过 MCP 明确识别。
- 股权代持迹象明显但无确证。
- VIE 架构或复杂离岸结构。
- 大额未了结诉讼或仲裁。
- 行政处罚涉及行业准入资质。
- 知产出质比例接近或超过 50%。
- 用户要求用于正式投资决策。
