# 投资标的信息联网搜索查询模板

## 何时读取本文件

当已经确定需要联网搜索，并需要构造公司、行业、管理层、知识产权、融资或法律问题 query 时，读取本文件。

## 本文件影响什么决策

本文件决定公开搜索的查询词、组合方式和主题模板，确保联网搜索只补充 MCP 未覆盖的信息。

## 不适用场景

尚未完成 MCP 主链路、已有结构化数据足以形成初筛结论，或只是在输出报告时，不优先读取本文件。

> 本文件是 `legato-investment-target-screening` 的联网搜索查询模板库。
> 本 Skill 以 MCP 企业数据工具为主要数据来源，联网搜索为辅助补充。
> Agent 在需要联网搜索时应优先使用本文件中的模板构造查询，而非自行生成泛化查询。
> 模板中的 `{公司名}`、`{行业}`、`{year}`、`{法律问题}` 为占位符，运行时替换为实际值。

## 1. MCP 查询指引

以下指引说明不同 MCP 工具的典型查询场景和使用时机（完整调用顺序和工具映射见 `references/mcp-routing.md`）：

### 企业识别与核验

| 场景 | MCP 工具 | 说明 |
| --- | --- | --- |
| 通过名称或关键词找到企业 | `get_company_by_query` | 首次使用，输入公司名称或关键词 |

| 场景 | MCP 工具 | 说明 |

### 风险扫描

| 场景 | MCP 工具 | 红旗信号 |

## 6. 确定性标签在筛查报告中的使用示例

以下示例展示如何在筛查报告中正确标注确定性标签，区分不同来源的可信度：

### 示例 1：MCP 数据驱动的风险判断

```markdown
目标公司存在 1 条失信被执行人记录，执行标的金额为 500 万元[^1]。

[^1]: 企业工商与风险数据 - 高 | 失信被执行人记录 | 2026-05-25 | MCP 实时查询返回，显示该公司存在未履行的失信被执行记录
```

### 示例 2：联网搜索补充的监管信号

```markdown
该公司于 2025 年 8 月被上海市市场监督管理局处以罚款 20 万元[^2]。MCP 行政处罚数据中未检索到该条处罚记录，以下信息来自联网搜索。

[^2]: [上海市市场监督管理局行政处罚决定书](URL) - 中 | 行政处罚公告 | 2025-08-15 | 联网搜索补充，MCP 未覆盖；处罚内容为虚假宣传
```

### 示例 3：法律框架分析

```markdown
根据《公司法》相关规定，股权冻结期间不得办理股权转让登记[^3]。

[^3]: [《中华人民共和国公司法》及相关司法解释](URL) - 高 | 法规原文 | 2024-07-01 | 用于分析股权冻结对投资交割的法律影响
```

### 示例 4：联网舆情信号

```markdown
近期有媒体报道该公司涉及裁员和薪资纠纷[^4]。

[^4]: [媒体报道标题](URL) - 低 | 联网新闻 | 2026-04 | 仅作舆情动态信号，需结合 MCP 数据和正式渠道信息综合判断
```

### 示例 5：MCP 数据缺失时的处理

```markdown
目标公司的知产出质信息未能获取[^5]。

[^5]: MCP 知产引擎未返回出质数据 - 待确认 | 数据缺失 | 2026-05-25 | 无法判断知产出质状况，建议通过企业信用信息公示系统手动查询
```

### 示例 6：多重来源交叉验证

```markdown
目标公司信用状况如下：MCP 失信被执行人查询未发现记录[^6a]，但信用中国显示存在 1 条行政处罚信息[^6b]。

[^6a]: 企业工商与风险数据 - 高 | 失信被执行人查询 | 2026-05-25 | 未检索到失信记录
[^6b]: [信用中国行政处罚公示](URL) - 中 | 行政处罚公告 | 2026-03-10 | 联网搜索补充维度，MCP 行政处罚工具未覆盖该条记录
```

## 7. 搜索结果评估纪律

### 来源可信度排序

MCP 结构化数据 > 官方域名搜索结果 > 权威法律解读 > 开放搜索结果 > 新闻媒体

### 访问状态标注

对推荐的每个联网资源，标注其访问状态：

- [已验证可访问]：URL 可用，内容可读
- [需注册]：URL 可用但需免费注册
- [需付费]：需要订阅
- [页面不存在]：URL 已失效
- [未验证]：无法检查（如网络问题）
