# 投资标的筛查搜索策略

## 何时读取本文件

当投资标的快筛需要补充联网搜索、行业监管动态、舆情、公开披露、管理层背景或法律问题材料时，必须先读取本文件。

## 本文件影响什么决策

本文件决定 MCP 企业数据、法律知识库、联网搜索和结果交叉验证的执行顺序，以及联网结果如何补充而非替代结构化数据。

## 不适用场景

仅执行企查查 / 元典 MCP 主链路、只读取专家方法，或只是在生成报告时，不以本文件替代 MCP 路由和输出规则。

> 本文件是 `legato-investment-target-screening` 自带的搜索策略指引。
> 本 Skill 以 MCP 企业数据工具为主（qcc-company、qcc-risk 等），联网搜索为辅。
> 目标是把 MCP 数据与联网搜索有机结合，形成完整筛查视角。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 通用搜索顺序

本 Skill 的数据来源以 MCP 结构化查询为核心，联网搜索作为补充。Agent 必须按以下顺序执行，不得跳过 MCP 直接联网搜索：

### 第 1 步：MCP 企业数据查询

按 `references/mcp-routing.md` 中的「调用顺序与成本控制」执行 MCP 数据收集：

- 第一轮（必选，约 8 次调用）：企业识别 → 工商核验 → 股东结构 → 实控人 → UBO → 失信 → 风险快扫。
- 第二轮（建议，约 5 次调用）：管理层红线 → 知产概览。
- 第三轮（可选，按需）：融资历史 → 治理稳定性 → 法规检索。

MCP 数据是本 Skill 的主要数据来源，联网搜索不得替代 MCP 的结构化数据。

### 第 2 步：法律知识库按需查询

对 MCP 数据中发现的法律问题（如代持安排效力、VIE 架构合规性、对赌条款效力），使用元典法律知识库查询相关法规和案例：

- `yuandian_law_vector_search`：按语义检索法规
- `yuandian_rh_fg_search`：查法规正文
- `yuandian_rh_fg_detail`：获取法条详情

查询失败时，进入第 3 步联网搜索。

### 第 3 步：联网搜索补充

对 MCP 数据未覆盖的维度或需要最新动态的场景，使用联网搜索补充：

#### 行政处罚与监管动态

优先搜索以下官方域名：

```
"{公司名}" 处罚 OR 行政处罚 site:samr.gov.cn
"{公司名}" 经营异常 site:credit.gov.cn
"{公司名}" 失信被执行人 site:zxgk.court.gov.cn
```

#### 舆情与行业风险

```
"{公司名}" 舆情 OR 风险 {year}
"{公司名}" 诉讼 OR 纠纷 {year}
"{公司名}" 行业 监管 {year}
```

#### 法规解读与行业分析

```
"{行业}" 合规 OR 监管 {year}
"{法律问题}" 解读 OR 案例 site:legal500.com/guides OR site:lexology.com
```

#### 法律主题→优先来源路由

当 MCP 数据触发法律问题需要联网核验时，按以下路由选择优先搜索域名：

| 法律主题 | 优先搜索域名 |
| --- | --- |
| 公司法（设立、治理、股东权利） | flk.npc.gov.cn → court.gov.cn → pkulaw.com |
| 合同效力（代持、对赌、回购） | court.gov.cn → flk.npc.gov.cn → pkulaw.com |
| 知识产权（专利、商标、著作权） | cnipa.gov.cn → court.gov.cn |
| 证券监管（信息披露、上市合规） | csrc.gov.cn → sse.com.cn / szse.cn |
| 劳动用工（社保、劳动争议） | mohrss.gov.cn → court.gov.cn |
| 反垄断（经营者集中、市场支配地位） | samr.gov.cn → flk.npc.gov.cn |
| 外汇/跨境资金 | safe.gov.cn → pbc.gov.cn |
| 数据安全/个人信息 | cac.gov.cn → flk.npc.gov.cn |

### 第 4 步：开放搜索

以上步骤均不足时，使用开放搜索补充。但结论强度自动降级，输出中必须标注为"待确认"。

## 2. 各步骤失败后的处理链

| 失败情况 | 下一步动作 | 输出要求 |
| --- | --- | --- |
| MCP 企业识别失败 | 提示用户提供 USCC 或精确公司名称 | 不写"公司不存在"，写"未通过名称匹配到企业" |
| MCP 某项数据返回空 | 记录"未获取"，不编造 | 标注"MCP 未返回该数据" |
| MCP 工具调用失败（超时 / 错误） | 重试一次；仍失败则跳过并记录 | 标注"该数据获取失败，筛查可能不完整" |
| 元典法规检索无结果 | 进入第 3 步联网搜索 | 标注"法规库未检索到，以下解读基于联网搜索" |
| 联网搜索无官方域名结果 | 使用开放搜索 | 结论强度降级，标注"待确认" |
| 开放搜索仅找到新闻或媒体 | 仅作为动态信号 | 结论标为"待确认"，不可单独支撑风险判断 |
| 来源之间存在冲突 | MCP 数据优先，联网搜索分别列明 | 无法收口时标为"不明确"或"需人工复核" |

## 3. 时效性前置判断

启动筛查任务时，Agent 应先判断该风险维度属于哪一级时效敏感度。

| 敏感度 | 典型领域 | 来源有效期窗口 | 行为要求 |
| --- | --- | --- | --- |
| 极高 | 行政处罚、经营异常名录、失信被执行人名单 | 实时/近实时 | MCP 数据为最新快照；联网搜索需说明查询日期；如 MCP 数据日期较久远，提示数据可能已变化 |
| 高 | 股权变更、法人变更、重大诉讼、股权冻结 | 1-3 个月 | 注明数据日期；如有重大发现建议交叉验证 |
| 中 | 知识产权、融资记录、招聘信息 | 3-12 个月 | 标准数据日期意识；注明数据覆盖范围 |
| 低 | 基本工商信息（成立日期、注册资本、经营范围） | 相对稳定 | 时效性不是关键问题，但仍应注明来源日期 |

对「极高」敏感度领域，报告中必须包含：

> 行政处罚和信用信息变化频繁。以下筛查数据基于截至 [日期] 的 MCP 快照和联网搜索，可能未反映最新变化。建议在正式投资决策前重新查询最新数据。

## 4. 联网搜索优先域名

本 Skill 联网搜索时应优先限定以下官方域名：

| 域名 | 覆盖范围 | 搜索价值 |
| --- | --- | --- |
| `samr.gov.cn` | 国家市场监督管理总局 | 行政处罚、经营异常、反垄断处罚 |
| `credit.gov.cn` | 信用中国 | 失信信息、行政处罚、经营异常 |
| `zxgk.court.gov.cn` | 中国执行信息公开网 | 失信被执行人、被执行人、限制消费 |
| `wenshu.court.gov.cn` | 中国裁判文书网 | 裁判文书（需关注文书公开范围限制） |
| `pbc.gov.cn` | 中国人民银行 | 行政处罚、支付牌照信息 |
| `csrc.gov.cn` | 中国证监会 | 行政处罚、监管措施、IPO 审核 |
| `cbirc.gov.cn` | 国家金融监督管理总局 | 行政处罚、监管措施 |

搜索顺序：先限定官方域名搜索，无结果再使用开放搜索。

## 5. MCP 与联网搜索的分工原则

| 维度 | MCP 数据角色 | 联网搜索角色 |
| --- | --- | --- |
| 企业工商信息 | 主要来源，结构化、可信赖 | 不需要 |
| 股权结构与实控人 | 主要来源 | 不需要（MCP 已覆盖） |
| 失信/被执行人 | 主要来源 | 补充验证（官方域名交叉核验） |
| 行政处罚 | 主要来源 | 补充 MCP 未覆盖的近期处罚 |
| 知识产权 | 主要来源 | 不需要 |
| 融资历史 | 主要来源 | 不需要 |
| 行业监管动态 | 不覆盖 | 主要来源 |
| 法律框架与合规分析 | 不覆盖 | 元典法规库 + 联网搜索 |
| 舆情与新闻 | 不覆盖 | 主要来源 |
| 最新重大事件 | 可能滞后 | 联网搜索为唯一来源 |

## 6. 搜索链路示例

以"筛查某科技公司投资标的风险"为例：

1. **第 1 步**：MCP 查询 → `get_company_by_query` 识别企业 → `get_company_registration_info` 核验 → `get_shareholder_info` + `get_actual_controller` 股权穿透 → `get_dishonest_info` + `get_judgment_debtor_info` + `get_high_consumption_restriction` + `get_equity_freeze` + `get_business_exception` + `get_administrative_penalty` 风险快扫 → `get_personnel_dishonest` 等管理层红线 → `get_patent_info` 等知产概览。
2. **第 2 步**：发现股权代持迹象 → `yuandian_law_vector_search` 查"股权代持 效力"相关法规。
3. **第 3 步**：联网搜索 `"公司名" 处罚 site:samr.gov.cn`、`"公司名" 舆情 OR 风险 2026`。
4. **第 4 步**：如以上搜索不足，开放搜索补充。
