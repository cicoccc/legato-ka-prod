# 输出与引用规则（德国投资税务合规）

## 何时读取本文件

当准备输出德国投资运营风险初筛、税务 / 用工 / 董事责任结论或正式报告时，必须先读取本文件。

## 本文件影响什么决策

本文件决定聊天摘要与正式 Markdown 报告结构、德国法来源引用、风险等级、证据缺口和人工复核提示。

## 不适用场景

仅做官方来源入口核验、搜索 query 构造或内部任务拆解时，不以本文件替代搜索策略。

> 本文件是 `legato-germany-investment-tax-compliance` 自带的输出与引用规则。
> 目标是确保德国投资税务场景下的所有法律结论有可定位来源、精度可控、时效性可追溯。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 输出层级

Legato 产品中，本 Skill 输出固定为"聊天摘要 + 强制正式 Markdown 交付物"：

1. 聊天窗口简短 Markdown 摘要，只帮助用户快速理解方向。
2. 正式 Markdown 报告交付物，是研究任务的强制交付面。

Markdown / Word 导出由产品交付物功能承担。

内部工具协议：

- 首轮研究任务必须使用 `write_file` 生成完整 Markdown 交付物，再使用 `present_artifacts` 展示。
- 同一研究的后续修改、补充、纠错、扩写或结构调整，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续对话如果是无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物，再使用 `present_artifacts` 展示。
- 纯解释既有交付物中某段含义、且不新增结论、依据、结构或行动清单时，可以只在聊天窗口回答。
- 工具名只用于 Skill 内部执行协议、eval 和研发说明，不得进入用户可见报告。

## 2. 聊天窗口回答

聊天窗口回答要短，帮助用户快速判断结果方向。

默认结构：

```markdown
## 简要结论
-

## 关键依据 / 风险
-

## 正式报告
- 已生成正式 Markdown 交付物，并已展示。

## 限制与复核
-
```

要求：

- 先给答案，再讲依据。
- 只放关键结论和限制。
- 不把完整研究报告塞进聊天消息。
- 不展示内部工具、供应商、配置编号或函数名。

## 3. 来源层级

德国投资税务场景的来源层级定义：

| 层级 | 来源 | 使用规则 |
| --- | --- | --- |
| 高 | `gesetze-im-internet.de`（联邦法规原文）、`bundesfinanzministerium.de`（财政部 BMF）、`bundesfinanzhof.de`（联邦财政法院 BFH 判例）、中德税收协定原文、EU 指令/规章 | 可支撑决定性税务结论 |
| 中 | Steuerberater（税务顾问协会）、WTS/国际所税务实务文章、Chambers/ICLG 德国税务指南、OECD 指南、BMF 行政指引 | 可辅助框架、实务和事实判断 |
| 低 | 行业报告、媒体快讯、财经资讯 | 只作动态信号 |
| 待确认 | 未返回可定位片段、版本不明、来源冲突或事实缺失 | 必须提示人工复核 |

与 `official-source-verification.md` 的衔接：该文件第 4 节"推荐官方来源入口"中已核验的 URL（BMAS、BMF、gesetze-im-internet.de、中德税收协定页面）属于"高"层级来源，可直接引用。该文件第 1 节已核验口径中的修正后写作口径，必须作为 Skill 输出的基准口径。

## 4. 确定性标签

报告中的每条税务信息必须标注确定性等级：

| 标签 | 含义 | 适用来源 |
| --- | --- | --- |
| [法规原文] | 直接引用法条或法规原文 | 高层级：gesetze-im-internet.de 法规文本、BFH 判例、BMF 正式指引 |
| [权威解读] | 来自 Chambers/ICLG 德国税务指南、OECD 指南、BMF 行政原则 | 中层级：高质量税务指南和官方行政指引 |
| [一般评论] | 来自 Steuerberater 文章、国际所税务 client alert、行业分析 | 中/低层级：律所实务文章、媒体 |
| [待验证] | 来自 AI 生成、论坛、无法确认来源，或未找到官方核验 | 无来源或低可靠来源 |

## 5. 精度纪律

德国税法精度要求极高，必须严格遵守：

- 源文件说 "kann" / "may" / "könnte"，不得转述为 "muss" / "requires" / "must"。
- 源文件说 "in der Regel" / "typically" / "grundsätzlich"，不得转述为 "immer" / "always" / "stets"。
- 源文件说 "bis zu" / "up to"，不得转述为 "genau" / "exactly"。
- 税率数值必须引用官方法规或 BMF 指引原文，不得凭记忆或二手来源写税率。
- 税收协定条文必须区分"上限税率"和"实际适用税率"，不得混淆。
- 阈值、门槛、期限必须注明法规条文出处和生效日期。
- 多个来源说法不一致时，分别列明，不揉成折中结论。

## 6. 脚注格式

优先使用 Markdown 脚注：

```markdown
正文中的关键税务结论[^1]。

[^1]: [来源标题](URL) - 高 | 法规原文 | 2026-05-25 | 说明该来源支撑的具体结论
```

德国投资税务场景的脚注必须包含日期，因为税法时效性极强：

```markdown
[^1]: [EStG § 2 Abs. 1](https://www.gesetze-im-internet.de/estg/__2.html) - 高 | 法规原文 | 2026 | 所得税法第 2 条第 1 款定义
```

如果当前只取得行政指引但尚未取得更高层级法条原文：

```markdown
[^2]: [BMF VWG Verrechnungspreise 2024](URL) - 高 | 行政指引 | 2024-12-12 | BMF 转让定价行政原则；该指引为行政解释层级，非立法文本
```

如果来源为高质量税务指南：

```markdown
[^3]: [Chambers Germany Tax Guide](URL) - 中 | 专业指南 | 2025 | 用于搭建问题框架；关键税率和门槛仍需官方来源核验
```

如果来源为国际所税务实务文章：

```markdown
[^4]: [Baker McKenzie - German Tax Update](URL) - 中 | 实务解读 | 2025 | 近期立法变化的实务解读；核心规则仍需官方法规核验
```

## 7. 必须引用的内容

- 税率、附加税率（如 Gewerbesteuer Hebesatz）、有效税率计算。
- 税收协定条文（如中德税收协定 Art. 5 PE、Art. 10 股息、Art. 11 利息、Art. 12 特许权使用费）。
- 阈值、门槛（如 GmbH 最低注册资本、VAT 注册门槛、最低工资）。
- 申报期限、合规期限、时效期限。
- 税收优惠条件和适用限制（如 § 8b KStG 参与免税、GewSt 清除规则）。
- 反避税规则（如 Hinzurechnungsbesteuerung、Außensteuergesetz、ATAD 转化规则）。
- 转让定价方法和文档义务（如 BMF 2024 VWG Verrechnungspreise）。
- "当前有效""最新变化""已生效/草案"等时效判断。
- 主管机关、办理路径、材料要求。
- 多法域比较中的德国侧关键差异。
- 依赖二手来源或弱来源的判断。
- `official-source-verification.md` 第 1 节已核验的修正口径。

## 8. 不得引用的内容

- 只有文件名、知识库名、网页标题，但没有可定位原文片段。
- 搜索摘要不足以支撑结论的内容。
- 推测性常识补全（如"德国税负很高"这类笼统表述）。
- AI 生成答案本身。
- 过时的税率或门槛（与 `official-source-verification.md` 已核验口径冲突的旧数据）。

### 引用真实性底线

以下内容属于 LLM 模板占位符或幻觉产物，不得出现在任何引用、脚注或来源标注中：

- `[Valid]`、`[Effective]`、`[Current Date]`、`[现行有效]`、`[有效]`
- `[Body]`、`[X]`、`[Name]`、`[Date]`、`[数量]`、`[客户名称]`
- 未经实际检索验证的 URL 或域名（不得编造看起来合理但未实际访问的来源地址）
- 无对应脚注实体的纯数字标记（如 `[1]`、`[2]` 但文末无对应脚注条目）

引用必须来自实际检索命中或知识库召回的结果。不确定来源是否真实存在时，标为 [待验证] 而非伪造引用。

## 9. 德语术语处理

- 对德国税法关键术语，必须保留德语原文，第一次出现时附中文/英文解释。
- 示例：Körperschaftsteuer（公司所得税）、Gewerbesteuer（营业税/贸易税）、Einkommensteuer（个人所得税）、Umsatzsteuer / Mehrwertsteuer（增值税）、Abzugsteuer（预提税）、Betriebsstätte（常设机构）。
- 不得将德语税法概念机械翻译后直接使用，需确认概念对应关系。

## 10. 冲突处理

来源冲突时：

- 分别列明冲突来源。
- 比较效力层级（法律 > 行政指引 > 法院判例 > 学术/实务文章）、发布时间、适用范围和具体事实。
- 如果仍无法收口，标为"不明确"或"需德国税务顾问确认"。

不要把冲突来源揉成一个看似确定的折中结论。

## 11. 翻译和原文

- 关键术语必须保留德语原文或英文对照。
- 用中文交付时，第一次出现原文术语后应解释含义。
- 不把机器翻译或非官方译文当作无风险原文。
- 法规条文引用应优先使用 gesetze-im-internet.de 的官方文本。
