# 中国法规案例研究 Skill

> 版本：v0.5  
> 日期：2026-06-06  
> 状态：productized  
> 运行模式：v0.1 固定 `agent`

## 定位

本 Skill 是 Legato 知识助手中的中国法研究兜底能力，用于需要以中国法律法规、规范性文件、监管材料或中国案例作为决定性依据的研究任务。

它提供的是研究、初筛、依据定位、案例口径整理、风险分级和正式 Markdown 报告，不替代律师最终法律意见、诉讼代理意见或合同红线修订。

## 适用场景

- 查中国法律、法规、规章、司法解释、监管规则或政策口径。
- 定位法条、确认法律依据、判断现行有效性或适用范围。
- 查中国案例、裁判规则、权威案例、典型案例或相似案例。
- 做中国法合规路径、风险分析、监管要求、行动清单。
- 基于用户挂载资料、企业内部资料和中国法源生成正式 Markdown 研究报告。

## 不适用边界

- 主问题是境外法、跨境投资、海外市场准入或多法域比较，应转出海法律与合规研究或更窄专家 Skill。
- 主任务是合同起草、合同审查、红线修订、诉讼文书或争议策略。
- 主任务是企业工商尽调、财务税务计算、政策宣传稿、翻译或排版。
- 用户要求最终法律意见时，只能输出研究结论、依据、限制和人工复核提示。

## 最低输入

最低可启动输入：

- 法域为中国大陆，或问题可合理判断为中国大陆法。
- 任务类型：法条检索、案例检索、合规路径、监管口径、风险分析或报告。
- 用户事实：主体、行业、行为、交易、争议或监管问题的基本描述。

信息不足时不调用即时追问工具；基于明确假设继续初筛，并列出会改变结论的关键缺口。

## 研究执行顺序

1. 先判断主任务是否属于中国法研究，而不是合同、尽调、诉讼或纯翻译。
2. 明确研究对象：法条、案例、监管口径、合规路径、风险分级或报告。
3. 按 `Reference 使用路由` 读取本任务必需的 reference，先确定主路径、MCP / 来源分工、引用规则和正式报告模板。
4. 按任务分型选择主路径：用户限定本次挂载资料 / 项目资料时先检索项目资料；法条问题先查法规；案例问题先查案例；最新监管动态再补官方联网核验。
5. 法条问题不默认查案例；案例问题不默认铺开所有法规。
6. 对最新政策、监管公告、办事指南或资料缺口，用联网公开来源补充核验；联网检索只表达地域、语言、复杂度、可信源要求、域名 / 时间约束和来源优先级，具体搜索工具选择遵循系统级搜索工具选择策略。
7. 输出前按引用规则检查可定位片段、来源层级和结论强度。
8. 涉及监管主题时按主题路由到对应主管机关；涉及具体省市区县时补充地方政府、地方法院或对应主管机关来源。
9. 输出时区分”已核验依据””案例口径””项目资料事实””待人工复核”；高风险、时效不明、来源冲突或无可定位片段时，必须降低结论强度。

## 任务分型

| 类型 | 主路径 | 交付重点 |
| --- | --- | --- |
| 法规探路 | 语义检索相关制度和规范层级 | 法规地图、适用范围、需继续定位的条文 |
| 法条定位 | 精确检索法规名、条号、关键词 | 条文、效力、适用条件、引用片段 |
| 案例探路 | 按事实场景找相似裁判口径 | 裁判规则、支持强度、差异事实 |
| 案例定位 | 按案由、法院、省份、案号或标题收窄 | 案例详情、可引用片段、类案限制 |
| 合规路径 | 法规 + 监管材料 + 项目资料组合 | 操作步骤、责任主体、材料清单、风险点 |
| 法律报告 | 多来源整合 | 执行摘要、依据、分析、行动清单、引用 |

## 运行约束

- `mode = agent`。
- 不调用 `ask_user_question`。
- 不使用旧 `query_kb`。
- 不自行选择知识库、文件夹或文件范围；检索范围由前端挂载和研究包配置决定。
- 内部工具名只允许出现在运行配置、研发排查或 eval 中，不进入用户可见报告。
- 首轮研究任务必须使用 `write_file` 生成完整 Markdown 交付物，并使用 `present_artifacts` 展示；不得因问题较窄、仅查法条、仅查案例或只需简短答复而省略交付物。
- 同一研究的后续修改、补充、纠错或结构调整，必须使用 `edit_file` 更新既有交付物，再使用 `present_artifacts` 展示。
- 后续对话如果是无关新研究，必须重新使用 `write_file` 创建新的 Markdown 交付物，再使用 `present_artifacts` 展示。
- 不固定底层输出路径或文件名模板，由产品运行时处理落盘位置。

## 引用与复核规则

- 所有引用必须有可定位原文片段；没有片段不得挂引用编号。
- 中国法法律结论应尽量包含法规 / 规则名、条号或章节、原文片段或 URL、日期和支撑点。
- 输出前过滤 `[Valid]`、`[Current Date]`、`[X]`、`[有效]`、`[现行有效]`、无脚注实体的纯数字标记等伪引用。
- 法律法规、司法解释、案例、监管材料应分别说明来源层级和支撑点。
- 来源冲突时，按效力层级、时间、适用范围和具体事实解释，不强行收口。
- 涉及重大合规后果、行政处罚、刑事风险、诉讼胜负、合同重大权利义务时，必须提示人工复核。

## 输出要求

聊天窗口只放简短摘要，包含结论、关键依据、风险 / 限制和交付物状态；聊天摘要不能替代正式交付物。法条定位、案例定位、简单法规检索、合规初筛等首轮研究任务都必须生成完整 Markdown 交付物。

正式报告必须先读取并使用 `assets/china-law-report-template.md`，不得把完整报告堆进聊天消息，也不得用 `references/output-and-citation-rules.md` 中的摘要结构替代正式报告模板。同一研究的后续修改用 `edit_file` 更新原报告；无关新研究新建交付物。
