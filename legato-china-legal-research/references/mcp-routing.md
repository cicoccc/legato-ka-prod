# 中国法研究路由规则

> 本文件说明中国法研究任务如何先按专家方法分型，再映射到产品运行时的内部数据源和工具。

## 何时读取本文件

- 开始任何中国法规案例研究任务时。
- 需要判断先查法规、案例、企业法律事实数据、项目资料还是联网公开来源时。

## 本文件影响什么决策

- 决定主任务分型、检索路径、内部数据源分工和追加检索条件。

## 不适用场景

- 不用于生成报告正文、引用脚注或用户可见来源说明；这些规则见 `output-and-citation-rules.md`。

## 1. 总原则

- 先判断主任务类型，再选择检索路径。
- 每轮先选一个主路径，不默认全量调用法规、案例、联网和企业数据。
- 法条依据优先于案例。
- 案例只在用户需要裁判口径、相似事实判断或法条无法解释适用边界时进入。
- 普通案例只作兜底，不作默认入口。
- 中国法决定性结论优先使用法条、法规、案例原文、官方材料或可定位片段。
- 用户可见输出只使用“中国法律法规库”“中国案例库”“企业法律事实数据”“本次挂载资料”“联网公开来源”等名称。

## 2. 主任务类型

| 类型 | 判断信号 | 专家主路径 | 追加路径 |
| --- | --- | --- | --- |
| 法条依据确认 | “有哪些法定条件”“是否必须”“处罚是什么”“定义是什么” | 先建立法规地图，再锁定法条详情 | 必要时补项目资料或官方公开来源 |
| 条文精确定位 | 已知法规名、条号、制度名、关键词 | 直接定位法规和条文详情 | 需要解释适用边界时再补探路 |
| 案例口径判断 | “法院通常怎么判”“类似情况如何认定” | 按事实场景寻找相似裁判口径 | 命中后获取案例详情 |
| 结构化案例定位 | 已知案由、法院、省份、案号、标题、权威案例 | 先查权威 / 典型案例 | 权威不足且确需样本时查普通案例 |
| 项目资料限定研究 | “只基于本次资料”“先看项目资料”“内部既有口径”“挂载资料中有没有” | 先检索本次挂载资料 / 项目资料 | 用户允许或资料不足时再补法规、案例或官方公开来源 |
| 国内合规操作 | “主管机关”“材料”“备案”“审批”“流程” | 先确定法律义务和监管对象 | 再查官方办事指南 / FAQ |
| 监管动态追踪 | “最近”“最新”“新规”“草案”“是否生效” | 先核验法规时效 | 再查最新官方动态 |

## 3. 内部工具映射

以下内部名称只用于执行配置，不进入用户可见输出。

| 任务目标 | 推荐内部工具 | 用户可见来源名 | 说明 |
| --- | --- | --- | --- |
| 法条探路 | `yuandian_law_vector_search` | 中国法律法规库 | 不确定该看哪部法、哪几条时，用语义检索建立法规地图 |
| 法条定位 | `yuandian_rh_ft_search`, `yuandian_rh_ft_detail`, `yuandian_rh_fg_search`, `yuandian_rh_fg_detail` | 中国法律法规库 | 已知法规名、条号、制度关键词时定位法规和法条详情 |
| 关键词收窄 | `yuandian_rh_ft_search`, `yuandian_rh_fg_search` | 中国法律法规库 | 已有明确制度关键词后的结构化收窄 |
| 案例探路 | `yuandian_case_vector_search` | 中国案例库 | 按事实场景寻找相似裁判口径 |
| 案例定位 | `yuandian_rh_qwal_search`, `yuandian_rh_ptal_search` | 中国案例库 | 按案由、法院、省份、案号、标题等线索定位案例 |
| 案例详情 | `yuandian_rh_case_details` | 中国案例库 | 获取案例详情和可引用片段 |
| 引用复核 | `yuandian_hall_detect` | 内部引用复核能力 | 复核法规、法条、案号、案例名等引用 |
| 项目资料检索 | `query_legato_ragflow(query_text, top_k)` | 本次挂载资料 / 项目资料 | 只检索前端本轮限定范围；不得自行选择知识库、文件夹或文件 |

## 3.1 项目资料优先规则

当用户明确要求“只基于本次资料”“先看项目资料”“内部既有口径”“从挂载资料里找”或同类意图时，先使用 `query_legato_ragflow(query_text, top_k)` 检索当前前端限定范围。

- 如果用户说“只基于本次资料”，不得主动补法规、案例或联网来源；如资料不足，在输出中说明“本次挂载资料未召回足够可定位片段”，并列出可能需要人工补充的材料。
- 如果用户说“如果不够再补中国法依据”，先检索项目资料；资料不足或需要外部法源支撑时，再进入法规或案例路径。
- RAGFlow 返回结果必须有可定位片段才能进入引用；只有文件名、知识库名或摘要不足以支撑结论。
- 输出中把该来源称为“本次挂载资料 / 项目资料”，不得暴露 `query_legato_ragflow`。

## 4. 何时停止

可以停止：

- 已拿到决定性法条原文和当前有效状态。
- 已拿到能支撑结论的法规详情或法条详情。
- 已拿到权威案例或案例详情，足以解释裁判口径。
- 用户限定只基于本次项目资料，且已说明已召回片段或资料不足状态。
- 新一轮检索只会重复已有结论。

不能停止：

- 只有法规名，没有条号、原文或适用范围。
- 只有实务文章，没有法条、法规或官方材料。
- 案例题只拿到标题，没有案号、法院、裁判日期或裁判要旨。
- 动态问题只拿到过时材料，未核验最新状态。
- 用户要求先看项目资料，但尚未检索本次前端限定范围。

## 5. 禁止路径

- 用户只要法条依据时，不自动调用案例数据源。
- 用户只问条文原文时，不先做宽泛语义检索。
- 结构化案例问题不先走宽泛案例语义探路。
- 未得到可定位原文片段时，不伪造引用编号。
- 中国法律法规库不可用时，不用联网普通文章替代决定性法条原文。
- 只有弱来源时，不写决定性结论。

## 6. 企业信息

企业法律事实数据仅在中国法问题涉及具体企业法律事实时使用，例如：

- 企业涉诉文书。
- 行政处罚。
- 被执行 / 失信。
- 知识产权或年报中与法律判断有关的事实。

如果任务主轴是企业画像、主体资信或交易对手尽调，应交由专门企业数据 Skill 或研究包配置处理，而不是由本 Skill 承担。
