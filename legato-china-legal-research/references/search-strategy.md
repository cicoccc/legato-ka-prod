# 中国法研究搜索策略

## 何时读取本文件

当中国法研究需要检索法规、案例、监管材料、官方口径或补充联网核验时，必须先读取本文件。

**前置依赖**：如涉及法条解释争议、法律结论推导或论证强度评估，应先读取 `references/legal-reasoning-methods.md` 确定分析框架。如涉及诉讼程序、特殊案由或程序转换，应先读取 `references/litigation-procedure-guide.md`。

## 本文件影响什么决策

本文件决定元典 MCP、案例检索、官方法规库核验和联网搜索的执行顺序，以及何时降低结论强度。

## 不适用场景

用户只要求解释已经给出的条文、案例或报告内容，且不需要新增检索时，不以本文件作为输出规则。

> 本文件是 `legato-china-legal-research` 自带的搜索策略指引。
> 目标是把"元典 MCP 优先、官方来源次之"的抽象原则翻译为 Agent 可逐步执行的具体搜索链路。
> 本 Skill 独立打包时不得依赖上级目录、兄弟目录或外部共享文件。

## 1. 主路径选择与搜索顺序

每个子问题必须先按 `references/mcp-routing.md` 判断主任务类型，再进入对应主路径。不得把“法条检索 -> 案例检索 -> 联网搜索”作为所有问题的固定顺序。

### 路径 A：项目资料优先

用户明确要求“只基于本次资料”“先看项目资料”“内部既有口径”“挂载资料中有没有”时，先使用 `query_legato_ragflow(query_text, top_k)` 检索当前前端限定范围。

- 用户要求“只基于本次资料”时，RAGFlow 是唯一主路径；资料不足时说明覆盖不足，不主动外扩。
- 用户允许“不够再补中国法依据”时，先走 RAGFlow，再按问题性质补法条、案例或官方联网核验。
- 只有可定位片段可以进入引用；文件名、知识库名或摘要不足时只能说明“未召回足够材料”。

### 路径 B：法条 / 法规主路径

法条依据确认、条文精确定位、定义、条件、处罚、程序、备案审批等问题，先使用中国法律法规库。

- **语义检索**（`yuandian_law_vector_search`）：不确定该看哪部法、哪几条时，用语义检索建立法规地图。
- **关键词检索**（`yuandian_rh_ft_search`、`yuandian_rh_fg_search`）：已知法规名、条号、制度关键词时，用关键词检索精确定位。
- 命中后获取法条详情（`yuandian_rh_ft_detail`、`yuandian_rh_fg_detail`）。
- 只有在用户需要裁判口径、相似事实判断或法条无法解释适用边界时，才追加案例检索。

目标：拿到决定性法条原文和当前有效状态。

### 路径 C：案例主路径

用户主问题是“法院怎么判”“有没有相似案例”“指导性 / 典型案例”“按案由、法院、省份、案号找案例”时，直接进入案例路径，不先做宽泛法条探路。

- **语义检索**（`yuandian_case_vector_search`）：按事实场景寻找相似裁判口径。
- **结构化检索**（`yuandian_rh_qwal_search`、`yuandian_rh_ptal_search`）：按案由、法院、省份、案号、标题等线索定位案例。
- 命中后获取案例详情（`yuandian_rh_case_details`）。
- 如案例结论需要解释法律依据，再补充法条详情。

目标：拿到权威案例或案例详情，足以解释裁判口径。

### 路径 D：监管动态 / 官方口径核验

监管动态、最新政策、办事指南、FAQ、草案和时效状态问题，应在法规库核验基础上补充官方联网核验：

优先域名（按重要性排序）：

| 域名 | 对应机构 | 典型内容 |
| --- | --- | --- |
| `flk.npc.gov.cn` | 全国人大法律法规数据库 | 法律、法规、司法解释、规范性文件原文 |
| `court.gov.cn` | 最高人民法院 | 司法解释、指导性案例、审判动态 |
| `spp.gov.cn` | 最高人民检察院 | 检察解释、典型案例 |
| `mofcom.gov.cn` | 商务部 | 外商投资、经营者集中、贸易法规 |
| `samr.gov.cn` | 市场监管总局 | 反垄断、市场监管、行政处罚 |
| `pbc.gov.cn` | 中国人民银行 | 金融监管、外汇管理、支付结算 |
| `safe.gov.cn` | 国家外汇管理局 | 外汇管理、跨境资金 |
| `customs.gov.cn` | 海关总署 | 进出口、关税、海关监管 |

#### 主题 -> 监管机构路由

同一中国法域内，先按主题选择主管机关，再构造联网检索意图：

| 主题 | 优先官方 / 自律来源 | 典型检索目标 |
| --- | --- | --- |
| 公司 / 证券 / 上市 | `csrc.gov.cn`、交易所官网、`npc.gov.cn` | 证券监管、上市规则、信息披露、公司法条文 |
| 私募基金 / 资管 | `amac.org.cn`、`csrc.gov.cn` | 登记备案、自律规则、资管监管 |
| 银行 / 保险 / 金融控股 | `nfra.gov.cn`、`pbc.gov.cn` | 金融监管规则、监管处罚、业务准入 |
| 外汇 / 跨境资金 | `safe.gov.cn`、`pbc.gov.cn` | 外汇登记、跨境收付、资金池 |
| 劳动 / 社保 / 用工 | `mohrss.gov.cn`、地方人社部门 | 劳动合同、社保、工时、用工政策 |
| 环保 / ESG / 环评 | `mee.gov.cn`、地方生态环境部门 | 环评、排污、环保处罚、碳相关政策 |
| 数据安全 / 网安 / 算法 | `cac.gov.cn`、`miit.gov.cn`、`npc.gov.cn` | 数据出境、网安审查、算法备案、个人信息保护 |
| 知识产权 | `cnipa.gov.cn`、`court.gov.cn` | 专利、商标、著作权、司法解释 |
| 税务 | `chinatax.gov.cn`、财政部官网 | 税收政策、申报、发票、优惠 |
| 反垄断 / 市场监管 | `samr.gov.cn`、`npc.gov.cn` | 经营者集中、垄断协议、行政处罚 |

#### 地名 -> 地方来源路由

当用户问题包含具体省、市、区县或项目所在地时，应补充地方政府、地方法院或对应主管机关来源。

规则：

1. 从用户问题中提取地名，例如深圳、上海、北京、重庆、山东、广州、无锡。
2. 结合主题选择地方来源类型：地方政府门户、地方法院、地方市场监管、人社、生态环境、税务等。
3. 地方规则不得覆盖上位法；报告中应说明地方口径与中央规则的关系。
4. 如果地方来源无结果，不得写成该地无规则，应说明本轮未找到可定位地方来源。

#### 中国法主题来源包构造规则

每轮联网核验应先构造“主题 -> 主管机关 -> 地方来源”的紧凑来源包：

1. 从主题路由表选择 2-5 个中央官方 / 自律来源。
2. 如用户问题含地名，再选择 1-3 个对应地方政府、地方主管机关或地方法院来源。
3. 若主题同时涉及中央规则和地方执行口径，报告中必须区分“上位法依据”和“地方执行口径”。
4. 若官方来源不足，可补充高质量律所、学术或专业机构文章，但只能用于理解实务，不单独支撑决定性法律结论。
5. 来源包只表达检索目标和域名 / 机构约束；具体调用哪个联网搜索工具由系统级搜索工具选择策略决定。

### 路径 E：高质量指南补充

官方来源无结果或覆盖不足时，在以下高质量中文法律来源中搜索：

- 最高法/最高检公报、指导性案例
- 权威法律网站（如中国法学学术期刊数据库、北大法宝等公开页面）
- 知名律所实务文章（金杜、中伦、方达、汉坤、竞天公诚等）
- 走出去智库（CGG）等专业机构文章

这些来源用于搭框架、做比较和识别需要官方核验的关键点，不替代官方来源。

### 路径 F：开放搜索

以上步骤均不足时，使用开放搜索补充。但结论强度自动降级，输出中必须标注为 [待验证]。

## 2. 各步骤失败后的处理链

| 失败情况 | 下一步动作 | 输出要求 |
| --- | --- | --- |
| RAGFlow 检索无结果 | 如果用户限定只基于项目资料，则停止外扩；如果用户允许补法源，则进入法规、案例或官方核验路径 | 说明本次挂载资料未召回足够可定位片段 |
| 元典 MCP 法条检索无结果 | 换语义检索或扩展关键词，再试一次 | 不因单次检索失败就放弃法规查找 |
| 元典 MCP 法条检索确实无覆盖 | 进入路径 D 官方联网核验 | 记录"元典法条库未覆盖该主题" |
| 元典 MCP 案例检索无结果 | 换案由或事实场景关键词，再试一次 | 不因单次检索失败就放弃案例查找 |
| 元典 MCP 案例检索确实无覆盖 | 进入路径 D 官方联网核验，或在确需样本时说明普通案例覆盖不足 | 记录"元典案例库未覆盖该主题" |
| 官方域名 404 或改版 | 换同机构其他官方域名，再试一次 | 不因单个域名不可用就放弃官方来源 |
| 官方域名限定搜索无结果 | 不写"该规则不存在"，进入路径 E | 标注"官方来源未检索到，以下结论基于二手来源" |
| 高质量指南无覆盖 | 进入路径 F | 结论强度降为 [待验证] |
| 开放搜索仅找到媒体或行业资讯 | 仅作为动态信号 | 结论标为 [待验证]，不可单独支撑法律结论 |
| 来源之间存在冲突 | 比较效力层级、发布时间、适用范围 | 无法收口时标为"不明确"或"需人工复核"，不把冲突来源揉成折中结论 |

## 3. 时效性敏感度分级

启动研究时，Agent 应先判断该法律领域属于哪一级时效敏感度，并在整个研究过程中保持对应的警惕度。

| 敏感度 | 典型领域 | 来源有效期窗口 | 行为要求 |
| --- | --- | --- | --- |
| 极高 | 制裁与出口管制、反垄断新规与申报标准修订、数据安全/个人信息保护（修法中）、外汇管制政策 | 3-6 个月 | 在执行摘要中加醒目时效警告；推荐用户直接查阅官方最新来源；明确说明指南可能已过时 |
| 高 | 税法修订、劳动法修订、外资准入负面清单、证券监管新规 | 6-12 个月 | 注明所有来源的发布日期；关键结论建议交叉验证至少两个来源 |
| 中 | 公司法、知识产权、合同法实务、物权法 | 1-3 年 | 标准来源日期意识；使用指南时注明年份 |
| 低 | 民法典基本原则、宪法、刑法总则、法律体系结构 | 相对稳定 | 时效性不是关键问题，但仍应注明来源日期 |

对「极高」敏感度领域，报告的执行摘要和结论限制中必须包含：

> 本领域规则变化频繁。以下信息基于截至 [日期] 的来源，可能未反映最新变化。建议直接查阅官方最新来源或咨询专业律师。

## 4. 引用复核

在完成主要检索后，应使用引用复核工具（`yuandian_hall_detect`）复核报告中引用的法规名称、法条号、案号、案例名等引用，确保引用准确性。

输出前还必须过滤 `[Valid]`、`[Current Date]`、`[X]`、`[有效]`、`[现行有效]`、无脚注实体的纯数字标记等伪引用。

## 5. 停止条件

可以停止：

- 已拿到决定性法条原文和当前有效状态。
- 已拿到能支撑结论的法规详情或法条详情。
- 已拿到权威案例或案例详情，足以解释裁判口径。
- 新一轮检索只会重复已有结论。

不能停止：

- 只有法规名，没有条号、原文或适用范围。
- 只有实务文章，没有法条、法规或官方材料。
- 案例问题只拿到标题，没有案号、法院、裁判日期或裁判要旨。
- 动态问题只拿到过时材料，未核验最新状态。
